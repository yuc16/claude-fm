## Script

好，欢迎回到 Claude FM。今天这期我们来聊一份非常有分量的研究报告。

这篇文章的英文标题是 Anthropic Economic Index report: Economic primitives，翻译成中文是「Anthropic 经济指数报告：经济原语」，发表于二〇二六年一月十五日，距今大约五个月。

五个月在 AI 领域不算太长，但我需要帮你们建立一下时间背景。这份报告分析的是二〇二五年十一月中旬的真实对话数据，那个时候 Claude Opus 四点五 刚刚要发布、还没有发布。报告里很多数据和结论反映的是那个时间节点的模型能力，之后 Anthropic 推出了更强的模型，部分能力边界可能已经有所变化。但报告提出的分析框架和方法论，我认为在今天依然是最权威的参考之一。

这是 Anthropic 经济指数系列的第四份报告。主要作者是 Ruth Appel、Maxim Massenkoff 和 Peter McCrory，另外还有几位协同作者。他们做了什么？用 Claude 本身分析了 Claude 的一百万条真实对话——消费者侧来自 Claude dot ai，企业侧来自第一方 API 客户，采样时间是二〇二五年十一月十三日到二十日，整整一周。

我们为什么要花二十五分钟仔细听这份报告？原因有三。

第一，这是目前公开的、关于大语言模型真实使用模式，数据量最大、维度最系统的分析之一。不是实验室基准测试，是真实世界中真实用户每天在做什么的统计。第二，报告首次引入了「经济原语」这套五维框架，其中的一些发现是反直觉的，尤其是任务成功率这个变量，它进入计算之后会让生产力估算的数字大幅缩水。第三，报告配套发布了完整数据集，供所有外部研究者使用，这是罕见的开放姿态。

好，我们正式开始。我会按五个核心发现来讲。

第一个核心发现，什么是「经济原语」，为什么这套框架值得认真对待。

「原语」这个词来自计算机科学，英文是 primitive，意思是最基础的操作单元，没法再往下拆了。操作系统有原语，比如读写内存、进程调度；编程语言有原语，比如变量、条件判断、函数调用。Anthropic 借用这个词来形容他们对人机对话的五个基础测量维度，认为这些维度组合在一起，就能描述一次对话的「经济特征」，就像三原色可以调出所有颜色一样。

这五个维度分别是：任务复杂度、人类和 AI 的技能水平、使用场景、AI 的自主度，以及任务成功率。

测量方式非常有创意，他们让 Claude 自己去阅读匿名对话，然后回答一系列具体问题。比如，「如果不用 AI，一个有能力的专业人员大约需要多少时间来完成这个任务？」「要理解用户的这条提问，大约需要几年的受教育背景？」「Claude 有没有成功完成用户交代的任务？」这些问题的回答，构成了五个原语的量化数据。

你可能第一反应就是：让 AI 给自己打分，这靠谱吗？报告对这个问题有很诚实的讨论。他们说，单个指标可能有误差，不是精准到秒，但「方向准确」——方向准确的意思是，高的确实比低的高，趋势是对的。他们通过两种方式验证了这一点：其一，把 Claude 对任务时长的估计和真实平台上人类完成软件工程任务的实测时间做了对比，高度相关；其二，把 Claude 对人类提示所需教育年限的估计，和各职业的实际学历分布做了对比，结果也对得上。

这让我想到工程领域里常用的「代理指标」概念。你无法直接测量某些东西，但你可以找一个和它高度相关的可测指标来代替。Claude 对教育年限的估计，就是实际任务认知负荷的一个代理指标。不完美，但有用。

报告拿两类任务做了对比来展示这套框架的价值：软件开发，和个人生活管理。

软件开发：如果不用 AI，估计需要约三点三小时完成；结合 AI 大约十五分钟。提示和回复都需要约十三点八年教育水平才能理解。六十四%的对话属于工作使用。任务成功率约六十一%。

个人生活管理：不用 AI 大约需要一点八小时；结合 AI 也是约十五分钟。教育水平需求约九到九点四年。仅十七%属于工作使用。任务成功率高达七十八%。

两类任务的原语画像差异极大。软件开发需要更高技能、使用场景更工作化，但成功率反而更低——因为它更难。个人生活管理更简单，Claude 成功率更高。如果你不做这种细分，只是笼统地说「AI 使用量增加了多少」，你丢失的信息比保留的还多。

第二个核心发现，任务分布和使用模式的新变化。

报告发现，用户对 Claude 的使用高度集中。Claude dot ai 上，最常见的十个任务就占了所有对话的二十四%。企业 API 客户那边更集中，前十个任务占了三十二%，比上一份报告的二十八%又提高了四个百分点。

如何理解这个集中度？O*NET 是美国劳工统计局的职业信息数据库，包含了数千个不同的工作任务描述。Claude 的所有对话都被映射到了这些任务标签上。结果是：在 Claude dot ai 上能识别出超过三千个不同的工作任务，但光排名前十的任务就覆盖了几乎四分之一的使用量。

这种集中度反映了什么？说明 AI 目前能创造最大价值的场景仍然非常聚焦。排名第一的任务是什么？「修改软件以纠正错误」，也就是代码调试。在 Claude dot ai 上占了全部对话的六%，在 API 上更是一成。整体来看，计算机和数学类任务占了 Claude dot ai 对话的三分之一，占 API 流量的将近一半。

但有些趋势值得注意。教育和图书馆类任务在 Claude dot ai 上的占比从去年一月的九%上升到了十一月的十五%，稳步增长。越来越多的人把 Claude 当成学习辅导工具在用。同时，行政支持类任务在 API 端的占比明显上升，暗示企业正在把 Claude 用于自动化邮件处理、文档管理、日程安排这些后台流程。

然后是一个很值得关注的模式变化：自动化和增强的拉锯。

报告把使用模式分成两大类。自动化（automation），是指用户把任务完全交给 Claude 去做，基本不来回互动；增强（augmentation），是指人机协作，用户在整个过程中持续参与、迭代反馈。去年八月份的时候，自动化一度超过了增强，这是历史上第一次，当时引发了很多讨论，被解读为用户对 AI 的信任度跃升了一个台阶，越来越愿意把任务完全委托出去。

但到了十一月，趋势反转了。增强重新占到五十二%，自动化回落到四十五%。

为什么反转？报告认为一个重要原因是产品功能的变化——Claude 在这个阶段新增了文件创建能力、持久记忆、以及工作流定制功能，这些功能的设计导向是促进人机协作，而不是完全委托。这给我一个启示：自动化和增强的比例，不只是用户「信不信任 AI」的体现，它在很大程度上是产品设计导向的结果。工具设计成什么样，用户的行为就会朝什么方向走。

第三个核心发现，地理分布：全球停滞，美国加速。

报告引入了一个叫 AUI 的指标，全称 Anthropic AI Usage Index，Anthropic AI 使用指数。它的定义是：一个地区实际的 Claude 使用量，除以根据该地区劳动年龄人口规模预期的使用量。AUI 大于一意味着比预期用得多，小于一意味着比预期用得少。丹麦的 AUI 是二点一，意思是丹麦人用 Claude 的密度是其人口比例预测值的两倍多。

全球来看，AUI 和人均 GDP 的相关性非常强。人均 GDP 每提升一%，每人头的 Claude 使用量约提升零点七%。排在前面的国家是美国、印度、日本、英国和韩国，与我们的直觉一致。

关键发现在于：各国之间的差距从八月到十一月基本没有变化，既没有在缩小，也没有在扩大。低收入国家没有在追赶，高收入国家也没有在拉开。AI 在全球层面的扩散是静止的，而且不均等。

这就带来了一个令人担忧的隐含逻辑：如果 AI 真能大幅提升生产力，而它的使用量又和收入高度相关，那 AI 有没有可能进一步扩大国家间的发展差距，而不是缩小？报告没有正面回答这个问题，但数据已经指向了这个方向。

使用场景在全球层面也有明显的收入梯度。低收入国家的用户，学业使用比例更高；高收入国家的用户，工作使用和个人使用更常见。这和技术采用曲线的经典理论完全吻合：早期进入低收入市场的用户往往是技术精英，有明确的高价值需求；随着普及程度提高，使用场景才会多元化，扩展到更日常的个人生活领域。

在美国国内，情况则截然不同。各州的 AUI 差距在三个月里明显缩小，基尼系数（Gini coefficient），也就是衡量不均等程度的标准统计量，从零点三七降到了零点三二。这是一个相当大的变动方向。

如果按这个速度延续，报告估计美国各州的 Claude 使用量可以在两到五年内基本均衡。做个历史对比：电力、汽车、电话这些上一个世纪的重大技术，在美国各州完全扩散均衡花了大约五十年。两到五年是五十年的十分之一，快了将近十倍。

报告用了很严谨的统计方法来支撑这个估计，包括用各州劳动力结构作为工具变量，来纠正采样噪声带来的偏差。更保守的两阶段最小二乘估计是四到五年，仍然比历史技术扩散快了很多。当然报告也坦诚地说，这个估计只基于三个月的数据，不确定性很大，实际扩散可能更慢。

影响美国各州 Claude 使用量最大的因素是劳动力结构，而不是收入。一个州里从事计算机和数学职业的工人占比每提升一%，Claude 使用量每人头就高出零点三六%，仅这一个变量就解释了各州差异的将近三分之二。道理很直观：技术工人更早、更自然地把 AI 工具融入日常工作流程，他们知道怎么用、用在哪里、用来做什么。

第四个核心发现，任务视野与成功率的真实数字。

这部分我个人认为是报告里最有原创价值的地方，因为它把一个一直很抽象的问题——「AI 到底能做多复杂的任务」——变成了一组具体的可测数字。

先介绍一个背景概念，叫做「任务视野」（task horizon）。这个概念来自 METR，一个专门评估 AI 自主工作能力的机构。他们研究发现了一个规律：AI 在短任务上成功率高，在长任务上成功率低；随着模型越来越强，这个成功率随任务时长下降的曲线会变得越来越平缓。METR 用「在什么时长的任务上能保持五十%成功率」来定义每一代模型的「任务视野」，这个时长越长，说明模型越强。

这份报告第一次用真实对话数据，而不是精心设计的基准测试题，来量化类似的东西。

在第一方 API 数据里，成功率随任务时长的增加而下降。不到一小时的任务成功率约六十%，五小时以上的任务成功率约四十五%。把这条曲线往下延伸，五十%成功率对应的任务时长大约是三点五小时。

做个横向对比：METR 的软件工程专项基准测试显示，Claude Sonnet 四点五 对应的任务视野是约两小时，Claude Opus 四点五 是约五小时。真实数据得到的三点五小时落在这两个数字中间，量级是匹配的。这给了我们信心——这套基于真实对话数据的测量方式，确实在捕捉真实的能力边界，没有系统性地偏高或偏低。

Claude dot ai 的情况则完全不同。成功率随任务时长下降的幅度非常平缓，外推出来，五十%成功率对应的任务时长大约是十九小时。

为什么差了五倍多？关键在于多轮对话的结构。当你和 Claude 来回交互的时候，实际上是在把一个复杂任务拆解成很多小步骤，每一轮都提供反馈、纠正方向，允许 Claude 调整中间结果。这让整体成功率远比单次 API 调用高得多。

我来打个比方。单次 API 调用就好像让一个工人闭着眼睛走迷宫，从入口到出口一次搞定，走错了全盘重来。多轮对话则是每走几步就开一次灯看看，发现偏了立刻修正。任务越长，中间可能走错的地方越多，开灯的频率就越重要。

这对实际工作有非常直接的启示。如果你在用 API 构建自动化流程，处理估计需要超过三到四小时人工时间的任务，就要意识到成功率已经降到了五十%以下，你需要在流程里加入中间校验或者错误处理机制，不能指望一次性搞定。如果是交互式使用 Claude dot ai 来处理复杂问题，多轮迭代要远比一次性丢一个超长提示效果好，每隔几步让 Claude 总结一下当前进展再继续，能显著提高成功率。

还有一个关于教育年限和成功率的权衡，非常微妙但很重要。任务越复杂、需要的教育背景越深，成功率越低，但从 AI 获得的加速倍数也越高。具体数字是：需要高中以下教育水平的任务成功率约七十%，需要大学本科水平的约六十六%，大约低了四个百分点。但加速倍数从约九倍增加到约十二倍。所以即使高复杂度任务的成功率低一些，净生产力收益仍然更高——高教育水平的工作者，从 AI 里能获得更大的绝对收益。

第五个核心发现，AI 对职业的真实影响：有效覆盖率、技能位移与生产力。

这是报告最后也是最有分量的部分，把前面的所有分析都汇聚到了一个实际问题上：AI 到底在多大程度上、以什么方式影响具体的职业和工人？

先说「有效 AI 覆盖率」这个概念。

Anthropic 在之前的报告里就估计过，约三十六%的职业有超过四分之一的任务在 Claude 的使用数据里出现过，约四%的职业有七成五以上的任务被覆盖。但这套统计只是在看「有没有出现」，没有回答两个更关键的问题：被覆盖的任务是不是这个职业里最花时间的？Claude 在这些任务上的成功率是多少？

这份报告引入了「有效 AI 覆盖率」来把这两个维度纳入计算。算法是：把每个任务的 Claude 成功率，乘以这个任务占工人日常工作时间的权重，加权求和，得到一个估计：Claude 能够成功完成一个工人多少比例的工作量。

这个指标带来了一些出人意料的重排，我来讲几个具体的例子，这些例子比任何抽象描述都更有说服力。

数据录入员（data entry keyers）。他们有九项工作任务，Claude 只覆盖了其中两项。光看任务覆盖率，大概是二十二%，不算高。但关键在于：被覆盖的那项恰好是他们时间占比最大的核心工作——从源文档读取信息并录入系统。Claude 在这种任务上成功率非常高。所以有效 AI 覆盖率远高于单纯的任务覆盖率，这个职业是受 AI 冲击最显著的之一。

放射科医生。他们的核心工作是解读影像诊断和撰写报告，这两项都是高时间占比、高成功率的任务，尽管他们还有很多 Claude 做不到的事情，比如各种行政工作和实地检查。结果是有效覆盖率比单纯任务覆盖率高出很多。

微生物学家。Claude 覆盖了他们约一半的任务，听起来不少，但没有覆盖最花时间的那些：使用专门实验室设备进行实地研究。手不能离开实验台的工作，还是人类的地盘。有效覆盖率反而低于任务覆盖率。

这套框架的核心启示是：不能只看 AI 覆盖了多少任务，还要看覆盖的是不是工作里最核心、最花时间的部分，以及 Claude 在那些任务上到底有多靠谱。覆盖了一堆边缘任务却错过了核心任务，实际影响可能远比数字看起来小。

接下来是技能位移（skill displacement）的问题，这是报告里最令人深思的发现之一。

报告发现，Claude 使用的任务整体上比整个经济中的任务需要更高的教育年限。整个经济中，平均任务需要十三点二年教育；在 Claude 数据里出现的任务，平均需要十四点四年，对应大专学历左右。

这意味着什么？如果 AI 把这些相对高教育年限要求的任务从人类的工作内容中移除，留下来的那些任务平均教育年限会下降，整体上产生「去技能化」效应。

报告举了几个非常具体的例子，我来转述一下，因为具体案例比泛泛而谈要有冲击力得多。

旅行代理（travel agents）。Claude 能做「规划和销售行程套餐」（需要约十三点五年教育）和「计算旅行和住宿费用」（约十三点四年）；留给人类的是「打印或申请交通票据」（约十二年）和「收取交通和住宿款项」（约十一点五年）。AI 拿走了需要更高判断力的规划工作，留下了更机械的执行工作。净效果是去技能化，旅行代理的工作变得更低技能了。

物业经理（property managers / real estate managers）则是相反的故事。Claude 能做「维护销售记录」（约十二点八年）和「审核市场租金水平」（约十二点六年）；留给人类的是「与建筑公司谈判」「向银行申请贷款」「与董事会开会」——这些都需要更高的专业判断和人际能力。AI 替代了行政事务，把复杂谈判和利益方管理留给了人类。净效果是升技能化。

教师行业也很说明问题。AI 可以帮老师批改作业、给学生提供辅导、撰写课题申请、做研究综述——但没法替代在教室里当面给真实学生上课、维护课堂秩序。留下来的那部分工作，变成了整个职业的核心价值所在。

这套分析框架借鉴了劳动经济学家 Autor 和 Thompson 的最新研究。他们的理论预测是：如果 AI 替代了一个职业里的高技能任务，这个职业的工资可能下降、就业可能增加；如果替代的是低技能任务，工资可能上升、就业可能减少。按这个逻辑，旅行代理未来面临工资压力，物业经理反而可能更值钱。当然，这些都是在当前 Claude 使用模式不变的假设下成立的预测，现实会更复杂。

最后来看生产力估算的修正，这是整个报告最硬核的量化分析。

Anthropic 在之前的研究里估计，如果 AI 被广泛采用，有可能让美国劳动生产率每年提高一点八个百分点，持续十年。这是一个非常大的数字——美国过去十年劳动生产率的平均年增速大概只有一个百分点左右。

这份报告用新的数据重新算了一遍，并且纳入了两个之前没有考虑的因素，结果数字发生了显著变化。

第一个因素：任务成功率。之前的估算假设 Claude 每次都能成功完成任务，把所有时间节省直接加总。但实际上 Claude 的成功率不是百分之百，如果把每个任务的时间节省乘以对应的成功率概率，再加总，估计值从一点八个百分点降到了一点二个百分点（Claude dot ai 数据），API 数据进一步降到了约一点零个百分点。这个缩水大约是三分之一到四成，不可小看。

第二个因素：任务互补性。这个概念更复杂，但现实中非常重要。想象一下老师的工作：AI 帮她备课效率提高了三倍，但她在教室里给学生讲课的时间没有减少，这些面授时间成了整个工作流程的瓶颈。就算备课更快，一天也只有二十四小时，总课时数不会因此增加，生产力提升就受到了制约。

报告用了 CES 生产函数（常数弹性替代生产函数）来量化这个效应。在任务之间高度互补（也就是缺了哪一步都不行）的情况下，生产力提升可能只有零点七到零点九个百分点。如果任务之间可以自由替换（工人能把时间全部重新分配到 AI 加速最多的任务上，抛弃那些瓶颈任务），提升可能高达两点二到两点六个百分点。

最终的估算范围非常宽：从零点六到两点六个百分点都有可能，取决于两个核心假设——AI 的可靠性，以及工人能在多大程度上把工作时间重新分配到 AI 加速效果最好的任务上。

但即使是最保守的一个百分点，报告指出这已经相当于把美国劳动生产率增速拉回到九十年代末二十一世纪初的水平，那是美国历史上生产力增长的黄金时代，每年超过三%的整体增速。把劳动生产率抬高一个百分点，在宏观经济里已经是非常可观的数字。

好，现在来说说这些发现对你的实际工作有什么具体启示，以及要避开哪些坑。

如果你是工程师或技术从业者，直接用 Claude 做技术工作，有几个具体操作建议。

第一，把你的提示质量当成一个可以调的杠杆来认真优化。报告有一个核心发现：人类提示所需教育年限和 Claude 回复所需教育年限的相关系数超过零点九二，在国家层面和美国各州层面都高度一致。这意味着你输入越精准、越专业，你得到的输出也越专业。很多时候你觉得 Claude 给的答案太浅、太泛，问题不在 Claude，在提示本身缺乏上下文、缺乏专业术语、缺乏明确预期。

第二，拆分长任务是核心策略，不是权宜之计。API 场景下，超过三到四小时人工时间的任务，成功率会跌到五十%以下。如果你在用 API 构建自动化流程，遇到这种量级的任务，要么在流程设计时把大任务拆成小步骤，要么加入中间的人工校验环节，别指望单次 prompt 搞定所有事情。

第三，在 Claude dot ai 上用交互式多轮对话处理复杂问题，要比一次性丢一个超长提示效果好得多。每隔几步让 Claude 总结一下当前进展和下一步计划，然后再继续，这样你始终在给 Claude 提供方向修正的机会，成功率会显著高于一条到底。

如果你在企业里负责 AI 产品落地或者业务决策，有几点值得特别关注。

报告里关于 API 和 Claude dot ai 使用模式的差异非常重要。API 流量几乎三分之二是自动化模式，任务高度集中在编程和行政事务，用户给 Claude 的自主度相对低；Claude dot ai 则更多元，学习、创作、个人辅助、情感支持都有，用户给 Claude 的自主度更高。这意味着如果你在设计面向内部员工的 AI 工具，要预期员工的使用场景会远比你预想的更多元，不能只按「自动化执行器」来设计，要留出协作空间。

另外，在评估某个业务流程受 AI 影响的程度时，有效覆盖率是比任务覆盖率更有价值的指标。你要问两个问题：AI 覆盖的是不是这个流程里最花时间的环节？Claude 在这些环节上的实际成功率怎么样？这两个问题加在一起，才能给你一个靠谱的影响评估。

如果你在思考自己的职业发展，可以用这份报告提供的框架来做个自我分析。看你的职业核心任务有没有被 AI 高效覆盖；那些被覆盖的任务是你工作里最花时间的吗；被覆盖的任务比剩下的任务教育年限要求高，还是低——这决定了你是面临去技能化还是升技能化。

最重要的一个避坑提醒：不要把「任务被覆盖」等同于「工作被替代」。报告本身也强调了这一点。当一条 Claude 对话对应的是老师在备课，并不意味着老师的备课时间实际上减少了，也不意味着这位老师的工作岗位就消失了。覆盖率是必要条件，不是充分条件，实际替代还取决于组织如何响应、劳动力市场如何调整。

好，最后来总结全篇。

这份报告做了一件非常有价值的事：它把关于 AI 经济影响的讨论，从「感觉上会怎样」推进到了「数据上显示如何」。五个经济原语，加上有效覆盖率、任务视野、技能位移分析和生产力重新估算，构成了一套目前为止最完整、最可操作的分析框架。

核心发现总结：全球 AI 使用量与 GDP 高度相关，差距稳定，短期内不会收敛；美国各州正以历史上快十倍的速度走向均衡；把任务成功率纳入计算，生产力提升的估算从一点八个百分点降至约一点零个百分点，缩水近一半；AI 整体上覆盖了要求更高教育年限的任务，对大多数职业产生净去技能化效应，但也有岗位因此升技能化；真实 API 场景下的五十%成功率对应约三点五小时任务时长，和实验室基准测试结果的量级相符。

我个人认为，这份报告最令人深思的一点，不是某个具体数字，而是它展示出 AI 对经济的影响是极其不均匀的。不均匀体现在地域上，体现在职业上，体现在任务类型上，也体现在使用模式上。这种不均匀不是暂时的过渡期现象，而是 AI 作为一种工具的内在属性——它天然地对特定类型的任务特别有效，对另一些任务效果有限。

理解这种不均匀，远比追着某个平均值跑要有价值。因为真正的机会和风险，都藏在这种不均匀里面。

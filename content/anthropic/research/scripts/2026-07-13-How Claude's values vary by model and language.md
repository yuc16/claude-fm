## Script

How Claude's values vary by model and language，Claude 的价值表达如何随模型和语言变化。这篇文章发表于二〇二六年七月十三日。今天是二〇二六年七月十九日，也就是说它才发布六天，不存在时效性过期的问题。它处在一个很有意思的阶段，Claude 已经进入四点六、四点七这一代，大家不再只问模型会不会写代码、会不会做题，而是开始追问一个更微妙的问题，同样一个 AI 助手，为什么有时候像一个鼓励型搭档，有时候像一个严厉的审稿人，有时候又像一个谨慎过头的合规同事。

这期值得听，是因为它把一个平时很难量化的东西，变成了可以观察、可以比较、甚至未来可能可以调参的对象。这个东西就是模型回答里体现出来的价值倾向。这里先强调一下，Anthropic 说的 values，中文我们可以翻译成价值，或者价值取向，但它不是说 Claude 内心真的相信什么。它指的是 Claude 在输出里表现出来的规范性考虑，比如诚实、谨慎、温暖、尊重用户偏好、减少伤害、追求准确等等。换句话说，不是模型有灵魂，而是模型的行为让用户感受到某种立场。

我们每天和大模型打交道，经常会遇到没有标准答案的问题。比如，你要不要跳槽，要不要跟朋友摊牌，团队技术路线该不该重构，创业项目是不是该砍掉。对于这种问题，模型不可能只靠事实回答。它要决定是先安慰你，还是直接指出你的盲点。它要决定是顺着你的思路帮你完善方案，还是提醒你这个方案可能伤害别人。它要决定是给出一个干净利落的结论，还是不断说我不确定、这里还需要更多信息。所有这些选择，都会让回答带上一种价值色彩。

Anthropic 这篇文章的核心问题就是，Claude 到底在什么情况下表达什么价值？这些价值会不会随着模型版本变化？会不会随着语言变化？如果一个用户用英语问同一个商业计划，另一个用户用印地语或者俄语问，Claude 给出的反馈气质会不会不同？如果会不同，这种不同是合理的文化适配，还是训练数据和评测覆盖不均带来的偏差？这就是今天这篇研究想往前推进的一步。

在这之前，Anthropic 做过一项叫 Values in the Wild 的工作，分析了七十万条匿名的 Claude 对话，识别出 Claude 回答中超过三千种不同价值。这个结果听起来很震撼，但也带来一个问题，三千多个标签太多了。你不可能每次模型迭代都拿三千多个价值逐项比较，也不可能跟产品经理说，新模型在第三百七十二个价值上上升了，在第一千八百个价值上下降了。信息太碎，反而看不出结构。

所以这篇文章做了一件很像数据降维的事情。想象你有一张巨大的城市地图，上面有三千多个兴趣点。你如果一个一个看，当然最精确，但你会迷路。更有用的做法，是先找出几条主干道，看看这个城市大体是南北走向还是东西走向，哪些区域靠近商业中心，哪些区域靠近居民区。Anthropic 在这里做的，就是把三千多个价值压缩成少数几条价值轴。每条轴的两端不是绝对对立的好坏，而是两组经常呈现此消彼长关系的价值倾向。

文章最后提炼出了四条关键轴，而且这四条轴在控制了任务、话题和用户表达的价值以后，能解释 Claude 价值变化中百分之十五的差异。百分之十五听起来不算大，但对于真实开放对话来说，已经不小了。因为真实对话的差异非常复杂，用户问心理咨询、写代码、商业计划、法律风险、小说润色，差别本来就巨大。能在排除这些因素以后，还看到稳定的价值维度，说明这里捕捉到的不是随机噪音，而是某种模型行为的结构性特征。

这四条轴我们一条一条讲。第一条是顺从和谨慎。顺从这一端，指的是 Claude 更愿意配合用户，尊重用户偏好，肯定用户想法，帮助用户往前推进。谨慎这一端，指的是 Claude 更主动考虑风险，提醒可能的伤害，给出负责任的引导。举个工程师更熟悉的例子，如果你让模型帮你写一个抓取网站数据的脚本，顺从型回答可能会说，当然可以，我们可以用某个库，先写一个基本版本。谨慎型回答可能会先问，你有没有权限，是否遵守服务条款，会不会造成过高请求压力。两者都可能是有价值的，但场景不同。你在合法内部数据迁移时，过度谨慎会让人烦；你在灰色场景里，过度顺从又可能真的出事。

第二条是温暖和严谨。温暖这一端包括积极表达、鼓励、关怀、幽默、不给用户制造羞耻感。严谨这一端包括准确性、透明度、纠错、要求证据、挑战假设。比如你把一段架构设计发给 Claude，说帮我看看。温暖型回答可能会先说，这个设计已经很清晰，你把关注点拆得不错，下面我帮你找几个可以加强的地方。严谨型回答可能直接说，这里有一个关键假设没有成立，你对峰值流量的估计缺少证据，而且故障恢复路径不完整。很多时候我们既想要温暖又想要严谨，但在实际回答里，这两种倾向会有权衡。尤其是在简短对话中，多说鼓励的话，就少了篇幅做详细批判；多做冷静纠错，用户感受到的情绪支持就会少一些。

第三条是深度和简洁。深度这一端，Claude 会展开解释，展示推理过程，补充细节，指出复杂性。简洁这一端，Claude 更倾向于只做用户要求的事，少扩展，直接给答案。工程师应该很熟悉这个差异。你问一个命令怎么写，有时候你只想要一行命令，不想听半天原理。可你在做系统设计评审时，如果模型只给结论，不解释背后的权衡，你又会觉得它不靠谱。所以深度不是永远更好，简洁也不是偷懒。关键是模型能不能理解你现在需要的是外卖式答案，还是白板讨论式答案。

第四条是坦诚和执行。坦诚这一端，Claude 会更主动说明自己的不确定性，承认限制，提醒哪里可能错。执行这一端，Claude 会更像一个产出导向的助手，尽量给出 polished，也就是打磨好的、自信的、可直接使用的结果。这个轴特别适合解释我们使用大模型时的一种矛盾心理。你让它写一封商务邮件，你可能希望它别老说我不确定，而是直接给一个能发的版本。你让它判断一个医疗、法律、金融相关问题，你又希望它别装作太确定。坦诚和执行的张力，实际上就是 AI 助手在可靠性和可用性之间的平衡。

那 Anthropic 是怎么把这些轴做出来的呢？方法上很有意思。他们先从之前研究里识别出的三千三百零七个价值出发，人工把含义相近的价值聚类，得到三百三十九个高层价值。然后他们用一个保护隐私的分析工具，从 Claude 平台抽取了三十万九千八百一十五段对话。这些对话都有一个共同点，就是用户给 Claude 的任务带有主观性，也就是没有唯一正确答案。数据来自二〇二六年五月的两周时间，样本平均覆盖三个模型，分别是 Sonnet 四点六、Opus 四点六、Opus 四点七，也覆盖 Claude 平台最常见的二十种语言。大致上，每一个模型和语言组合都有约五千段对话。

接下来，他们让 Claude 去标注每段对话中这三百三十九个高层价值是否出现。注意，这里标的是 Claude 回答里表达了什么价值，也会同样识别用户表达的价值，以及对话的任务和话题。文章还提到，他们去掉了十八个在超过百分之八十对话中都出现的价值，比如有帮助、清晰、遵循指令。这些价值太普遍，如果不去掉，会淹没真正能区分不同对话的变化。最后，他们用降维方法，根据哪些价值经常一起出现，哪些价值经常此消彼长，把它们压缩成刚才说的四条轴。

这里有一个细节很关键，也是这篇研究比简单印象流更可信的地方。他们控制了任务、话题和用户表达的价值。为什么这很重要？因为如果不控制，你可能会把用户差异误判成模型差异。比如英语用户更常问法律合规，印地语用户更常问情感支持，那 Claude 用英语显得更严谨、用印地语显得更温暖，也许只是因为问题不同。Anthropic 的做法，是尽量比较在同类任务、同类话题、同类用户价值表达下，Claude 自己的回答倾向还会不会变化。当然这不能消除所有混杂因素，但至少说明他们意识到了这个问题，并且在方法上做了处理。

先看不同模型。文章比较了 Sonnet 四点六、Opus 四点六、Opus 四点七在四条轴上的平均位置。结论很有意思，也和很多用户的主观感受对得上。Sonnet 四点六最偏向顺从和温暖。它更常肯定用户的想法和工作，更容易用幽默、轻松、安慰的方式回应用户，不太带评判感。你可以把它想成一个情绪智商比较高、愿意陪你把东西打磨出来的搭档。比如你发一段写得不完美的产品文案，它可能先抓住亮点，说这个方向挺清楚，然后再温和地建议怎么改。

Opus 四点七则更偏向谨慎、严谨、深度和坦诚。它更可能主动提醒风险，挑战用户假设，坦率批评用户的工作，也更愿意展示自己得出结论的理由。同时，它也更可能承认限制，说某些地方它不确定，需要更多证据。这个画像很像一个资深 reviewer。你请它看设计方案，它不会只说不错，而会问你的容量规划证据在哪里，你的安全边界是什么，你有没有考虑滥用风险。对于需要高可靠性的任务，这是好事。但如果你只是想快速得到一个初稿，它可能显得有点啰嗦，甚至让人感觉不够配合。

Opus 四点六在文章里呈现出的特点，是相对更偏简洁和执行。它更容易直奔主题，留在用户请求的范围内，少做额外展开。这个差异也很实用。不是所有任务都需要 Opus 四点七那种深度质疑。有时候你只是要把一段 release note 改成更专业，或者把一段日志解释成可能原因，过多的价值提醒和自我限制反而降低效率。Anthropic 也说，这些模型间差异相对于不同对话之间的巨大差异来说是小的，但它们是有结构、可检测的。

这点很重要。不要把这篇文章理解成 Sonnet 永远温暖，Opus 四点七永远严厉。真实使用中，prompt、任务、上下文都会强烈影响模型。文章说的是在大量对话平均之后，能看到这些倾向。就像你说某个同事比较谨慎，并不是说他每句话都谨慎，而是在很多场景里，他更容易先想到风险。模型也是类似。

文章还提到，这些结果和 Anthropic 内部以及网上用户对模型性格的看法一致。用户会觉得 Opus 四点七更常 hedge，也就是对答案加限定，表达不确定性。Anthropic 员工也把 Opus 四点七描述为更透明、诚实、谦逊，把 Opus 四点六描述为更简洁。而 Sonnet 四点六在发布时就被描述为温暖、诚实、亲社会。现在这套价值轴能把这些主观印象用数据复现出来，说明它确实抓到了一些真实行为模式。

然后是语言差异。这个部分对中国听众尤其重要，因为我们很多产品都不是纯英语产品。你可能在做面向全球用户的 agent，或者你自己团队同时用中文和英文 prompt。文章说，Claude 在不同语言中表达的价值会有明显差异。不是说它翻译一下就完全一样，而是同样类型的请求，用不同语言表达，Claude 的回答气质可能不同。

为什么会这样？文章给了几个可能原因。首先，不同语言的训练数据数量和组成不同。英语数据通常更多，很多专业文档、学术文本、代码讨论、法律政策文本也集中在英语里。其他语言的数据量可能少一些，构成也不同。某种语言的训练数据如果更多来自日常对话、社交媒体、礼貌表达，模型学到的互动风格可能就更温暖。某种语言如果更多来自专业写作和技术文档，模型可能更严谨。其次，系统卡里的评测已经显示，Claude 在不同语言中的知识表现，以及处理敏感请求的方式，会有差异。价值表达差异可以看成同一个问题的另一个侧面。

具体结果是，Claude 在语言之间变化最大的是温暖和严谨这条轴，以及坦诚和执行这条轴。相对更稳定的是顺从和谨慎，以及深度和简洁。更细地说，在顺从和谨慎上，Claude 用阿拉伯语时最偏顺从，用英语时最偏谨慎。在温暖和严谨上，Claude 用印地语和阿拉伯语时最偏温暖，表现为礼貌语言、幽默、玩笑、肯定用户想法和工作；用英语和俄语时最偏严谨，表现为挑战假设、纠正细节、要求证据。在深度和简洁上，Claude 用英语时更偏深度，会改进和纠正细节；用阿拉伯语时更偏简洁。在坦诚和执行上，Claude 用荷兰语时更偏坦诚，比如承认自己的错误；用印尼语时更偏执行。

文章举了一个很直观的例子。如果两个人拿同一个商业计划找 Claude 反馈，一个人用印地语，一个人用俄语，他们可能会得到不同的感受。印地语用户可能听到更多鼓励和肯定，觉得这个计划挺有希望，只是需要完善。俄语用户可能收到更硬的质疑和证据要求，觉得这个计划问题很多。注意，底层事实可能没有变，但表达价值的方式变了，用户对方案质量的感知就会变。这对产品来说非常关键。模型不是只在传递信息，它也在塑造用户的信心、判断和行动。

不过，Anthropic 在这里非常谨慎。他们没有说哪种语言的表现一定更好，也没有说所有差异都是坏事。不同语言确实承载不同的会话规范。比如某些语言和文化里，礼貌铺垫更重要，直接批评可能显得冒犯；另一些语境里，直截了当指出问题反而被认为更专业。问题在于，模型现在已经在不同语言中给用户不同体验，但我们还不知道这些差异是用户想要的文化适配，还是训练不均衡导致的非预期偏移。要判断这一点，不能只靠英语团队拍脑袋，而要理解这些语言使用者自己的偏好。

我个人认为，这篇文章真正有价值的地方，不是告诉我们 Claude 哪个模型最暖、哪种语言最严谨，而是给了 AI 产品一个新的评估维度。过去我们评测模型，常常看准确率、拒答率、幻觉率、代码通过率、延迟、成本。现在 Anthropic 想补上一类更细的人机互动指标，也就是模型在开放对话中的价值轮廓。对于越来越多的 agent 产品，这个指标会变得非常重要。因为 agent 不只是回答问题，它会推动用户做决定，协调工作流，给人反馈，甚至在团队里扮演某种角色。

那这对我们实际工作有什么启发？先说模型选型。如果你在做一个面向普通用户的写作助手、学习陪伴、情绪支持、创意 brainstorming 产品，Sonnet 四点六这种更温暖、更顺从的倾向可能会让用户更愿意继续交流。用户拿来一篇文章，说我写得是不是很差，如果模型一上来就像审稿人一样说结构混乱、论据不足，用户可能直接关掉。但如果模型先承认努力，再指出可操作的改进，产品留存可能更好。

如果你在做代码审查、安全审计、合规检查、投资尽调、医疗信息辅助这类高风险或高精度场景，Opus 四点七的严谨、谨慎、坦诚就可能更合适。你不希望模型为了让用户开心而少说风险，也不希望它把不确定的判断包装成确定结论。比如你让模型审查一段权限控制代码，它如果很温暖地说整体不错，只提几个风格问题，却漏掉越权风险，那温暖就是负价值。这个时候你要的是它不怕冒犯你，敢说这里危险。

再说 prompt 设计。既然这些价值轴可以描述模型行为，我们就可以在 prompt 里更明确地指定想要的平衡。比如在设计系统提示时，不要只写你是一个有帮助的助手，因为有帮助太宽泛了。你可以写，在给出反馈时，先简短肯定已有工作，再直接指出关键风险，不要为了礼貌而弱化严重问题。或者写，用户在此场景中需要可执行结果，只有在存在重大不确定或安全风险时才展开说明限制。这样的提示，其实就是在温暖和严谨、坦诚和执行之间设置目标位置。

但这里有个坑。不要以为一句 prompt 就能稳定控制所有价值。文章最后也提到，未来一个重要方向是研究能不能可靠地 steering，也就是引导模型表达特定价值。可能通过 character training，也就是性格训练调整，也可能通过 system prompt，也就是系统提示词变化，然后用价值轴方法验证模型是否真的按预期移动。对我们做应用的人来说，最好也别只凭几条样例判断 prompt 成功。你需要收集一批真实或模拟对话，标注模型是否过度顺从、过度谨慎、过度冗长、过度自信，再看整体分布。

第三个启发是，多语言产品不能只做英语评测。很多团队习惯先用英文 benchmark 选模型，再把产品翻译到中文、日文、印尼语、阿拉伯语。可是这篇文章提醒我们，语言不是一个薄薄的表层。模型换一种语言，价值表达就可能变化。你在英语里测试到的安全边界、反馈风格、拒答倾向，不一定原样迁移到中文。尤其如果你的用户会用中文问职业建议、亲密关系、心理困扰、商业决策，这些都属于主观任务，模型的温暖、严谨、谨慎、坦诚会直接影响用户体验。

对于中国团队，我建议至少做三件小事。第一，建立中文自己的主观任务评测集，不要只测知识问答和代码。可以包含简历修改、绩效反馈、创业计划、团队冲突、学习规划、敏感但良性的安全咨询等。第二，同一批任务用中文和英文都跑一遍，比较回答气质，而不只是比较事实对错。第三，如果产品面向不同地区或不同语言用户，不要假设一个统一的系统提示就够了。你可能需要在不违背核心安全价值的前提下，做语言和文化层面的表达适配。

第四个启发，是在团队内部建立一种新的讨论语言。以前大家争论模型回答好不好，常常说得很抽象。有人说太啰嗦，有人说太冷，有人说太怂，有人说太冒进。价值轴给了我们更可操作的词。太怂，可能是在顺从和谨慎轴上过度谨慎。太冷，可能是在温暖和严谨轴上严谨有余、情绪支持不足。太啰嗦，可能是在深度和简洁轴上没有根据任务切换。太装，可能是在坦诚和执行轴上过度执行、自信包装太强。这样一来，产品、工程、研究、安全团队就能围绕同一套概念讨论，而不是互相猜情绪。

当然，也要看到这篇研究的限制。文章自己也说，他们现在测到了差异，但还不知道差异从哪里来。可能来自预训练数据，可能来自微调数据，可能来自不同语言数据数量和组成，也可能来自后训练中的性格塑造。四条价值轴只是指出应该去哪里看。要真正干预，还得把差异追溯到具体数据、训练阶段或上下文因素。

他们也还没有测这些价值差异对用户结果的影响。比如更温暖是否让用户更信任 Claude？更严谨是否提高决策质量？更坦诚是否降低误用，还是会让用户觉得模型不可靠？文章提到可以用 Anthropic Interviewer 之类工具询问用户的幸福感、对 Claude 的信任、决策质量，然后把这些结果和 Claude 表达的价值关联起来。这个方向很关键，因为价值差异本身不是终点，用户结果才是终点。如果一种差异不影响用户，优先级可能没那么高；如果一种差异会让某个语言群体持续得到较差建议，那就必须修。

还有一个值得我们警惕的点，价值轴不是道德评分表。温暖不是一定好，严谨不是一定好；顺从不是一定坏，谨慎也不是一定好。真正的问题是场景匹配。一个好的 AI 助手，应该像一个成熟的同事，知道什么时候鼓励，什么时候质疑，什么时候给短答案，什么时候展开推理，什么时候承认不确定，什么时候直接交付。我们不应该把模型训练成永远温暖，或者永远严格。我们应该让它能根据用户目标、风险水平、上下文和文化规范动态调整。

如果你来做一个企业内部的 AI 助手，可以怎么用这篇文章？我会建议把任务分成几类。第一类是低风险高频任务，比如会议纪要、文案润色、格式转换，这类任务可以偏执行、偏简洁，减少不必要的免责声明。第二类是创造性协作，比如产品命名、方案 brainstorm，可以偏温暖和顺从，让用户有探索空间，但也要在最后加一点严谨筛选。第三类是评审和决策，比如架构评审、事故复盘、采购建议，要显著提高严谨、深度和坦诚。第四类是安全、合规、隐私相关任务，要提高谨慎，并且明确风险边界。你甚至可以把这些价值轴写进内部 prompt 模板，让不同业务线按任务选风格。

如果你在做 agent，更要注意。因为 agent 会连续行动，价值倾向会被放大。一个过度顺从的 agent，可能会在用户给出模糊目标时一路执行，直到造成错误。一个过度谨慎的 agent，可能每一步都请求确认，最后用户不用了。一个过度简洁的 agent，可能不解释关键决策，难以审计。一个过度坦诚的 agent，可能不停暴露不确定性，让流程卡住。agent 的价值轮廓不只是回答风格，而是执行策略的一部分。未来我们可能会像配置温度、最大 token、工具权限一样，配置 agent 的谨慎度、批判度、解释深度和自我披露程度。

对评测体系来说，这篇文章也提示我们，不要只用单轮标准题。价值表达往往出现在开放式、多轮、带主观判断的对话里。你要测模型是不是过度迎合用户，就需要让用户提出一些有缺陷的计划，看模型会不会盲目肯定。你要测模型是不是过度严厉，就要给它一些新手作品，看它是否能指出问题但不打击人。你要测多语言一致性，就要把同一任务用不同语言自然表达，而不是机器翻译得很生硬。然后看模型在事实判断之外，情绪、风险、证据、限制这些维度是否一致且合适。

我个人还想补充一个中文语境下的观察。中文用户经常希望模型既懂人情又讲道理。比如职场沟通建议，如果模型太直接，会显得不懂中国组织里的委婉表达；如果模型太圆滑，又可能掩盖真实冲突。中文技术讨论也是这样。很多人用中文问架构问题时，希望模型先给结论，再给理由，不一定喜欢英文论文式长篇展开。可是当问题涉及安全、金融、法律时，又需要它足够严谨和谨慎。所以中文应用不能简单追求更像英语，或更不像英语，而要找到中文用户真正需要的平衡。

文章最后展望了几个方向。第一，追问价值差异从哪里来，能不能追溯到训练数据、训练阶段或上下文。第二，研究这些差异对用户意味着什么，比如信任、幸福感、决策质量。第三，弄清 Claude 的价值应该如何跨语言变化，这需要听取各语言使用者的观点。第四，研究除了语言和模型以外，还有哪些因素会影响价值表达，比如年龄、职业、地区等人口统计信号，或者用户文字里的语气、话题、风格。第五，测试能不能可靠地引导模型价值。第六，把价值画像纳入模型发布前评估和上线后监控，用来发现意外漂移，甚至和不遵守 Claude 宪法这类问题建立关联。

这里的 Claude 宪法，也就是 constitution，是 Anthropic 用来描述 Claude 应该体现哪些高层价值的原则集合。文章开头说得很清楚，任何文档都不可能预见每天数百万对话里会出现的所有价值情境。真正目标不是列出无穷规则，而是培养 Claude 在不同上下文中应用良好判断和稳健价值的能力。这句话我觉得是整篇文章的底层哲学。大模型越强，越不可能只靠规则表管理。我们需要能观察、能衡量、能反馈的系统，让模型在开放世界里表现出可解释的行为倾向。

总结一下，Anthropic 这篇文章做了三件事。它把 Claude 回答里三千多个价值标签，压缩成顺从与谨慎、温暖与严谨、深度与简洁、坦诚与执行四条轴。它发现不同 Claude 模型确实有不同价值画像，Sonnet 四点六更温暖顺从，Opus 四点七更严谨谨慎深入坦诚，Opus 四点六更简洁执行。它还发现 Claude 在不同语言中的价值表达会系统变化，尤其是在温暖与严谨、坦诚与执行上。

我个人认为，这不是一篇只对 Anthropic 内部有用的研究，而是在提示整个行业，模型的“性格”正在变成一类需要工程化管理的对象。对我们这些做软件、做 AI 产品的人来说，下一步不是问模型有没有价值观，而是问，在我的场景里，用户需要模型表达什么价值，哪些价值必须一致，哪些价值可以因语言和文化而调整，以及我有没有办法持续监控这种变化。今天这期就到这里。下次当你觉得某个模型太冷、太啰嗦、太会哄人，或者太爱警告时，不妨把它放到这四条轴上看一看，也许你会更清楚自己到底想改什么。

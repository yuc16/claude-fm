## Script

这篇文章的英文标题叫做 Anthropic Economic Index report: Learning curves，我把它翻译成"Anthropic 经济指数报告：学习曲线"。文章发表于二〇二六年三月二十四日，距今约三个月，研究的是今年二月份一百万条 Claude 对话数据，恰好和 Claude Opus 四点六 正式发布的时间重叠。

今天我专门拿出一期来讲这篇报告，是因为它触碰到了一个我觉得对所有做技术工作的人都非常关键的问题：在使用 AI 工具这件事上，经验积累到底有没有价值？越用越厉害，是一种真实存在的学习效应，还是自我感觉良好的幻觉？我看过很多人对这个问题的感性判断，但这是我见过的第一份用大规模数据系统回答这个问题的研究。

答案是：有效应，而且显著。

但在直接讲那个核心发现之前，我们先来搞清楚这个研究是怎么做的，因为方法论本身就很值得了解。

Anthropic 经济指数，英文叫 Anthropic Economic Index，简称 AEI，是 Anthropic 从二〇二五年开始推出的一个持续研究项目。它的目标，按 Anthropic 自己的说法，是帮助研究者和政策制定者尽早了解 AI 的经济影响，从而有充足的时间做出应对。

做这个研究面临一个根本性的难题：Anthropic 有海量的用户对话数据，但这些对话是私密的，不能直接读取或分析内容。那怎么在不侵犯隐私的前提下，搞清楚"人们在用 AI 做什么"？

他们的解法，是把每段对话映射到一个叫做 O*NET 的职业任务数据库。O*NET 是美国劳工部维护的一个非常详尽的数据库，里面记录了美国各种职业每天都在做哪些具体任务、需要什么程度的教育背景、平均时薪是多少。通过这种映射，Anthropic 能在完全不读对话文字内容的情况下，知道"这段对话大概对应什么职业的工作，这类工作在劳动市场上值多少钱"。

他们把这套分析框架叫做"经济原语"，economic primitives。核心变量包括几个：任务对应的职业时薪、任务所需的教育年限、交互模式是自动化还是协作增强、对话的最终成功率，以及一些描述人机分工的指标，比如谁主导、AI 被赋予多大的自主权。

这份最新报告是这个系列的第三期，研究的是今年二月五日到十二日这一周的数据，对比了二〇二五年八月和二〇二五年十一月的历史数据，形成了一个跨越约半年的追踪对比。

好，背景交代完了。报告分两大章节：第一章讲"与上期相比发生了什么变化"，第二章讲今天标题里说的"学习曲线"。我们先快速走一遍第一章，然后把更多时间留给第二章。

第一章的第一个发现是，Claude.ai 的使用场景变得更分散了。

什么叫更分散？你可能会问。我来用一个数字说明：在二〇二五年十一月，Claude.ai 上使用最频繁的前十种任务，占了全部对话流量的百分之二十四。到了今年二月，这个比例降到了百分之十九。也就是说，原本有一小撮"热门任务"在主导大部分流量，现在它们的垄断地位在减弱，更多样化的长尾任务开始进来了。

这背后有几个相互交织的原因，值得分开来说清楚。

第一个原因是：写代码这件事在"搬家"。大量代码相关任务正在从 Claude.ai 这个网页端迁移到第一方 API，尤其是 Claude Code——Anthropic 的代码智能体工具——在 API 端已经占了相当大的流量份额。Claude Code 的工作方式有一个特点：它把一个完整的编程任务拆解成很多小步骤，每个步骤作为独立的 API 调用分别处理，每一个小调用被标注为不同的子任务类型。所以虽然编程需求的总量在增长，但它被分散到了很多细粒度的子任务里，分布在各种任务类别里，反而让 API 端看起来更分散，而不是更集中。

第二个原因是用户结构的变化。课程作业类的对话，占比从百分之十九降到了百分之十二。这里有季节性因素——今年二月份的采样期间，部分国家的学生正处于寒假，学习相关使用量自然减少，据报告统计，在学生放假的国家里，课程类对话的降幅高达十二个百分点，而在正常上课的国家只降了五个百分点。同时，个人用途的对话从百分之三十五上升到了百分之四十二。报告里还提到，因为 Anthropic 在超级碗期间投放了广告，吸引了大量新用户注册，这批新用户往往是从最轻松随意的个人查询开始的，比如查个天气、问个体育比分、让 AI 帮你出个聚会菜单。

第一章的第二个发现是，Claude.ai 上任务的平均经济价值略有下降。

Anthropic 用一个非常具体的方式来衡量"任务的价值"：看做这个任务的职业，在美国劳动市场上的平均时薪是多少。举个例子：如果你用 Claude 帮你写一段 Python 脚本，对应的职业是软件工程师，时薪大约五六十美元；如果你用 Claude 查一下昨天 NBA 谁赢了，这个问题没有明确对应的高薪职业，经济价值相对就低。

按这个标准，Claude.ai 上任务的平均时薪从四十九点三美元降到了四十七点九美元。降幅不大，但方向是下降的。原因是体育赛事、天气查询、产品比价、家庭维修问题这类个人日常查询增多了，而编程类任务则更多转移到了 API 端。

但这里有一个非常重要的背景，我要特别强调：即便降了之后，四十七点九美元的平均时薪，仍然显著高于美国全国平均工资水平。Claude 整体上承担的依然是高附加值、高教育门槛的工作，这一点没有改变。报告把这个趋势描述为标准的"技术采用曲线"——任何技术在从早期用户扩散到大众用户的过程中，都会经历任务类型下移的阶段，这是普及的自然结果，不是能力的退步。

同期还有一些辅助指标也反映了类似趋势：Claude.ai 上对话里，用户提问所需的平均受教育年数从十二点二年降到了十一点九年，用户给 AI 的自主权略有上升，完成任务所需的估计时间也略有缩短。

第一章的第三个发现，关于地理分布上的不平等。这里有一个很有意思的分化：美国国内的差距在收敛，但全球层面的差距在扩大。

在美国国内，使用量最高的五个州，人均 Claude 使用量在全美总量里的份额，从二〇二五年八月的百分之三十降到了今年二月的百分之二十四。低使用量的州在快速追赶，差距在缩小。如果按这个速度推算，大约需要五到九年，美国各州之间才能达到人均使用量大致均衡的状态——注意，报告给了一个范围，五年或九年，取决于统计模型里是否加权。

但在全球层面，情况刚好相反。排名前二十的高使用量国家，人均 Claude 使用量占全球总量的份额，从百分之四十五上升到了百分之四十八。领先国家的优势在加速巩固，落后国家正在进一步掉队。

这个国内收敛、国际分化的对比，背后逻辑其实不难理解。美国国内，基础设施统一、语言统一、法规基本一致，普及靠口碑和推广就能扩散。但国家之间的壁垒——语言、网络基础设施、支付方式、监管政策——不是短期内能跨越的。AI 的好处，在这个阶段，确实更容易在已经领先的地方进一步积累。

好，第一章说完了。现在我们进入这期报告我认为最有价值的核心内容——学习曲线。

这章研究的核心问题，其实是两个相互关联的问题：第一，用户是否会根据任务的难度，有意识地选择合适的 AI 模型？第二，使用 Claude 时间越长的人，使用效果是不是真的更好？

我们先从模型选择说起，这部分有趣的数据可以为后面的学习曲线分析做铺垫。

Claude 有好几个档次的模型系列，从轻到重分别是 Haiku、Sonnet 和 Opus。简单理解：Haiku 最快最便宜，适合简单任务；Opus 能力最强最贵，适合复杂推理；Sonnet 是中间档，也是默认推荐的那个。如果用户是理性的，他们应该会把最重要、最复杂的任务带给 Opus，把简单任务留给 Sonnet 甚至 Haiku。

实际情况是这样吗？

数据说：是的，而且非常明显。

在付费版 Claude.ai 用户里，电脑和数学相关任务——也就是软件开发类的专业工作——有百分之五十五的对话使用了 Opus；而教育辅导类任务只有百分之四十五用了 Opus。细化到更具体的职业维度：软件开发相关任务里有百分之三十四用 Opus，家教辅导相关任务里只有百分之十二用 Opus。换句话说，在有模型选择权的付费用户里，Opus 被集中用在了更专业、更高价值的工作上。

更有意思的是，这个规律在第一方 API 用户里更加明显——API 用户对任务复杂度的模型选择响应，大约是 Claude.ai 用户的两倍。具体来说，在 Claude.ai 端，任务时薪每提高十美元，用 Opus 的比例大约多增加一点五个百分点；但在 API 端，同样的十美元对应了两点八个百分点的 Opus 使用率增加。

为什么 API 用户的模型选择更精细？报告的解释是：API 用户通常在搭建程序化的工作流，他们有非常强的动机去精细化地分配不同任务给不同模型——高价值、高复杂度的任务用 Opus 确保质量，低价值、高频次的任务用轻量模型控制成本。这是一种"元层面"的能力：知道什么时候需要动用最强的工具，什么时候不值得。

这里有个思考点我想停一下说一说。模型选择，这件事本身就代表了一种需要积累才能形成的使用技能。你得先用过足够多的任务，踩过足够多的坑，才能建立起"这种任务 Sonnet 就够了、那种任务非 Opus 不可"的直觉。这种直觉不是一开始就有的，它需要经验。这也为接下来要讲的学习曲线做了一个很好的引子。

好，现在正式来讲学习曲线。这是这期报告的标题所指的核心内容，也是我认为对你最有价值的部分。

Claude 第一个模型发布于二〇二三年三月，距离这份报告发布刚好三年。在这三年里，有从第一天就开始用的技术先锋，也有刚被超级碗广告吸引进来的第一次用户。这两批人的使用习惯和效果，差距有多大？

Anthropic 把用户分成两组做对比：高资历组，定义为注册 Claude 至少六个月以上的用户；低资历组，就是所有其他人。他们比较了这两组用户在各项指标上的差异。

结论出来，差异相当显著，而且在多个维度同时出现。

第一，任务难度不同。高资历用户在 Claude 上做的任务，对应的平均受教育年限高出约一年。注意，这里说的不是用户自己的教育程度，而是任务对应的行业需要多少年教育背景。换句话说，用 Claude 越久的人，倾向于用它攻克更复杂、门槛更高的工作，而不是停在简单的日常查询上。

第二，使用目的不同。高资历用户的个人日常查询少了百分之十，工作相关的使用多了百分之七。他们更倾向于把 Claude 当成工作工具，而不是消遣娱乐。

第三，使用方式不同。这是一个很有意思的发现。高资历用户更倾向于和 Claude 迭代协作，而不是单纯下指令让 Claude 全权搞定。报告把人机交互分成两大类：自动化，就是你一次性把任务扔给 AI，让它自主完成，人基本不在循环里；增强，就是你和 AI 来回打磨，AI 补充你的能力，但你保持主导，每一轮你都在审查和引导。高资历用户明显更多采用增强协作的方式。

这个发现其实推翻了 Anthropic 自己之前的一个假设。他们原本猜测，越资深的用户越可能倾向于自动化——毕竟更懂技术，更有能力搭建自动化流程，把 Claude 当 AI agent 来跑。但数据告诉他们：不对，最资深的用户反而是最喜欢和 Claude 来回迭代、协同工作的那群人。

第四，使用范围更广。高资历用户的使用分布更分散，前十类任务占他们总使用量的比例略低于新用户。他们用 Claude 处理了更多种类的工作，而不是总在做同一类事。

但最关键的发现，是第五条：成功率更高。

高资历用户的对话成功率，比低资历用户高出约五个百分点。

你可能会问，"成功率"这个指标是怎么定义的？报告里说，这是 Claude 对自己每段对话是否成功的自我评估。我知道这不是完美的指标，依赖 AI 评估 AI 自身，有它的局限性。但在一百万条对话的规模下，这个指标应该是有参考价值的，因为系统性偏差在两组用户之间是一致的。

现在关键的问题来了：这个五个百分点的差距，是真正的学习效应，还是别的原因造成的？

研究团队为了回答这个问题，做了三层统计控制，一层层地排除可能的混淆因素。

第一层，最简单的直接对比：高资历用户成功率高五个百分点。

第二层，控制任务类型。这一层的逻辑是：也许高资历用户成功率高，只是因为他们在做更容易成功的任务？为了排除这种可能，研究团队在相同的任务分类里对比高低资历用户。也就是说，不是跨任务对比，而是"同样是做企业财务分析和估值这个任务，高资历用户比新用户的成功率高多少"。控制了任务类型之后，差距从五个百分点缩小到三个百分点左右。这个缩小是合理的——高资历用户确实会选更难的任务，排除这个因素后，效应自然变小一些。但三个百分点的差距还在。

第三层，继续控制所用的模型、对话语言、用户所在国家、任务的具体类别等所有可测量的因素。把这些都控制住之后，高资历用户的成功率依然稳定地高出约四个百分点。

这四个百分点，是在排除了所有这些可能混淆的变量之后，还剩下的纯粹"经验效应"。它告诉你：不是因为高资历用户做的任务更简单，不是因为他们恰好在成功率更高的国家，不是因为他们选了更强的模型——在所有这些条件都相同的情况下，用得越久的人，对话越容易成功。

研究团队很坦诚地提出了两种可能的解释，没有粗暴地下结论说"这就是学习效应"。

第一种解释是存活者偏差加选择效应。使用六个月以上还在坚持用的人，很可能本来就是那些从 Claude 里得到了实实在在价值的人，所以才坚持下来了。那些用了一段时间觉得没什么用的人，早就不用了，不在这个样本里。另外，三年前就开始用 Claude 的早期用户，本来可能就是技术背景更强的人，比如专门研究 AI 的工程师——他们的高成功率，也许反映的是个人能力，而不是学习效应。

第二种解释是真正的边干边学。人们通过持续使用，确实学会了怎么更好地提问、怎么给 AI 更清晰的背景信息、怎么判断 AI 给的答案好不好、怎么在 AI 偏离方向时引导它回来。这种使用 AI 的元能力，随着经验积累而提升，导致成功率上升。

报告承认，现有数据无法完全区分这两种解释。但他们能做到的是，在统计上排除了"只是因为任务不同"这个最明显的混淆因素，而效应依然存在。这一点，是支持学习效应的有力证据。

报告里还有一个细节，我觉得特别生动，能帮你直观感受这两批用户的差异。他们统计了不同任务的"平均用户资历"——做这个任务的用户群，平均用 Claude 多久了。平均资历最高的任务包括：AI 研究、git 操作，也就是代码版本控制、修改学术论文手稿、创业融资相关的文档准备。平均资历最低的任务有：写俳句、查体育比分、为朋友的派对推荐食物。

这个对比画出了两幅截然不同的用户画像。一类是三年前就开始用 Claude 的技术先驱，他们把 Claude 用在 AI 研究、代码管理、学术写作这些需要专业深度的工作上，他们早已建立起了复杂的使用习惯。另一类是最近才被广告吸引进来的新用户，他们先从最轻松日常的事情开始——写个小诗、查个比分，这很正常，所有人都是从简单的地方入门的。

现在，让我来说说这份报告对你的实际工作意味着什么，以及你可以怎么用。

报告给出的最核心启示，我认为是这一条：在使用 AI 工具这件事上，经验有真实的复利效应，而这种复利是可以刻意去培养的，而不是等着它自动发生。

怎么刻意培养？我总结了几个具体的做法。

第一，主动挑战更难的任务。高资历用户最显著的特征之一，是他们倾向于把 Claude 用在教育门槛更高的专业工作上。这不是巧合，而是一种使用习惯。如果你总是只把 AI 用来做简单的事——改改格式、翻个句子、查个信息——你的学习曲线会非常平。真正能积累经验、提升成功率的，是把 AI 用去攻克那些对你来说有挑战性的任务。哪怕第一次失败，第二次失败，第三次逐渐找到门道，这个过程本身就是在建立你的 AI 使用能力。

第二，把协作迭代当作习惯，而不是一次性下指令。高资历用户最与众不同的行为模式，是他们更喜欢和 Claude 来回打磨，而不是发出一个长长的指令然后等结果。你可以把这个习惯理解为：把 Claude 当协作者，而不是执行器。具体来说，当 Claude 给出回答时，不要直接接受或者直接否定，而是认真看它的回答，找到哪里还不够好，用非常具体的反馈去引导它改进。"这段逻辑不太对，因为……你能重新考虑一下这部分吗"，这类迭代过程，本身就是在逼你更清晰地思考自己到底想要什么。

第三，刻意研究模型选择。报告数据很清楚地显示，API 用户对不同模型的任务分配比 Claude.ai 用户更精细，而且更精细的分配是有实际价值的。如果你在工作中使用 API 或者有模型选择的权限，值得认真思考一套属于你自己的模型路由策略：什么样的任务值得用 Opus，需要最强的推理能力；什么样的任务用 Sonnet 完全够用；什么样的高频、简单任务可以交给 Haiku。这不只是成本优化，它还能训练你对任务复杂度的系统性感知能力，这本身是一种很有价值的元认知。

第四，记录成功的提示词策略。很多人用 AI 工具的方式是随用随丢——每次从头想怎么问，每次提问效果好不好，过几天全忘了。但如果你把成功的提问结构记录下来，比如"对于这类代码审查任务，给 Claude 加上这几条背景信息效果最好"，或者"解释复杂概念的时候，让它先给类比再给细节，效果比直接讲细节好得多"，这些积累会让你的成功率稳步提升。就像程序员维护自己的代码库一样，好的提示词策略也值得沉淀下来，反复复用和优化。

第五，重视"元提示"能力，也就是怎么给 AI 设定背景、限制和期望的能力。高资历用户成功率更高，很大程度上可能来自于他们更善于让 Claude 理解他们想要的是什么、不想要的是什么。这是可以学习的技能：给出清晰的任务背景，明确说明期望的输出格式，在 AI 偏离方向时指出来并引导纠正，让 AI 知道你作为专业人士有哪些隐含的质量标准。这些"元层面"的沟通技巧，正是区分高资历用户和新用户的核心能力之一。

最后，我想谈一下报告里一个更宏观、也更让我觉得值得深思的议题：不平等。

报告在结论部分提到了一个经济学概念叫做"技能偏向型技术变革"。历史上，每次重大技术革命——工业自动化、计算机普及——都呈现出这样的模式：新技术往往会提高高技能工人的生产力，同时降低对低技能工作的需求，从而拉大两者之间的薪资差距，加剧劳动市场的不平等。

这份报告给了这个历史规律一个非常具体的机制，说明 AI 可能是如何重蹈这个覆辙的：早期采用者——往往是技术背景强、任务价值高的人——在使用 AI 时成功率更高，获得的生产力提升更大，从而在收入和竞争力上进一步领先。而大量化采用阶段涌入的普通用户，面对同样的 AI 工具，可能因为缺乏使用经验而获得的价值少得多。优势在自我强化，差距在扩大。

我个人认为，这是一个值得认真对待的警示。AI 工具本身是平等开放提供给所有人的，但"用好 AI"这件事本身，就是一种需要学习和积累的技能，而学习资源和学习意愿，并不是均匀分布在所有人之间的。

报告里有一句话让我觉得很有力量：早期采用者可能同时是受 AI 冲击最大的人，也是最能从 AI 中获益的人。这个"双向曝光"的现象——被颠覆，同时也被赋能——在软件工程师这个群体里体现得尤其明显。你们是 AI 最先深入渗透的职业之一，同时也是目前从 AI 中获得最多生产力提升的那批人。

Anthropic 做这个经济指数项目，目的之一就是尽早发出这类预警，让研究者和政策制定者有时间设计应对措施——比如 AI 素养教育、针对特定职业群体的技能培训。这是一项很有社会价值的工作，尽管距离真正能影响政策还有很长的路要走。

好，我们来收尾。

今天我们聊的是 Anthropic 经济指数系列的第三期报告，标题叫做"学习曲线"，发表于今年三月。报告的核心发现可以用三句话总结：Claude 的使用场景在持续多样化，任务平均价值微降但整体仍属高价值工作；老用户在同等任务下成功率高出新用户约四个百分点，这个差距无法被任务选择差异所解释；全球采用不平等在国内收敛，在国际层面却在扩大。

我个人认为，这份报告给所有 AI 从业者发出了一个清晰的信号：现在投入时间认真学习怎么用 AI，不是在做没用的事，而是在积累一种真实存在复利效应的竞争优势。

但我也要说，学习曲线不是自动弯曲的。它需要你主动去挑战更难的任务，主动迭代你的使用方式，主动记录什么有效、什么没效。只有这样，你才能走在那条向上倾斜的曲线上，而不是停留在平台期。

这就是今天的节目。我们下期见。

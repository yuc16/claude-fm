Anthropic Economic Index report: Cadences，中文可以翻译成，Anthropic 经济指数报告：节奏。这篇文章发表于二〇二六年六月二十六日。今天是二〇二六年六月二十八日，所以这是一篇非常新的报告，几乎就是刚刚出炉。它值得我们花二十多分钟来听，原因很简单：这不是一篇模型能力榜单，也不是一次新功能发布，而是 Anthropic 试图回答一个更大的问题，AI 到底是怎样一点点嵌进真实经济生活里的。

如果你是软件工程师，或者在公司里做 AI 应用，你可能每天都在问类似的问题：大家真的在用 AI 做什么？是写代码更多，还是写邮件更多？是工作日用，还是周末用？当一个人说他用 Claude 完成了一个任务，这个任务的最终产物到底是什么？是一个解释、一份文档、一段代码、一个网站，还是一个可以直接交付给老板或客户的东西？更进一步，人在多大程度上把判断权交给了 AI？这种交出去的行为，是让人更焦虑，还是更乐观？

这篇报告的标题叫 Cadences，也就是节奏。这个词选得很有意思。它不是在说某个静态切片，比如某一天有多少人用 Claude，而是在说 AI 使用背后有周期、有律动、有日常生活的痕迹。早上有人问新闻，傍晚有人问菜谱，凌晨有人问睡眠建议，工作日更多是邮件、营销文案、幻灯片，周末更多是情感支持、医疗问题、投资建议，到了美国报税截止日前后，税务相关请求突然暴涨。换句话说，Claude 的使用日志像一张经济活动的心电图，能看到人们工作和生活的脉搏。

这篇报告的背景，是 Claude 的使用形态在过去一年发生了很大变化。一年前，大多数 Claude 使用还比较像传统聊天，就是用户发一句，助手回一句，来回对话。可是随着 Claude Code 和 Cowork 这样的产品增长，越来越多会话不再是短短几轮问答，而是长时间运行的 agentic tasks，也就是带有自主规划和执行能力的代理式任务。Claude Code 可以理解为面向开发者的代理式编程工具，用户不只是问它某段代码怎么写，而是可能让它读项目、改文件、跑测试、修 bug。Cowork 则更像面向工作场景的协作界面，让 Claude 承担更长链条的任务。

这就带来一个测量问题。过去你只看聊天记录，大概还能知道用户问了什么、Claude 答了什么。但现在不够了。一个长时间任务里，AI 可能调用工具、生成多个文件、逐步修改结果，最后用户真正拿走的是一个可交付物。聊天 transcript，也就是文本转录，已经不能完整捕捉 AI 在经济中的作用。所以 Anthropic 在这次 Economic Index，也就是经济指数报告里，调整了数据管线。

他们做了几件事。第一，采样频率更高，可以看到小时级别的使用模式，而不是像之前报告那样主要抽取七天样本。第二，引入新的分类器，给每段会话的输出打标签，也就是判断 Claude 最终产出了什么 artifact。这里的 artifact 可以翻译成可交付物，或者会话产物，比如解释、文档、报告、代码、学术论文、应用、脚本、菜谱等等。第三，报告开始提供更细颗粒度的数据，把 chat 和 Cowork 这类 Claude conversations，以及第一方 API 的使用，在月度层面拆开看。这里第一方 API 指的是直接通过 Anthropic 自己接口路由的开发者流量，不包括第三方云平台，也不包括 Claude Code。

还有一个重要补充，就是他们开始做 Anthropic Economic Index Survey，也就是经济指数调查。光看使用日志，只能知道人们怎么用 Claude，却不知道人们心里怎么想。比如，用户觉得 AI 会不会改变自己的工作？会不会影响工资？会不会威胁职位？如果理想一点，他们希望 AI 在十年后塑造一个什么样的经济？这次报告把调查结果和 Claude 使用数据通过隐私保护方法连接起来，最终得到大约九千七百名受访者的关联样本。注意，这个样本不是普通大众代表性样本，而是 Claude 用户样本，而且还过滤掉了使用太少的人，所以它更像是重度或至少中度 AI 用户的观察窗口。

我们先看第一个核心点，AI 使用有非常明显的工作周节奏。报告发现，在 chat 和 Cowork 会话中，被分类为个人用途的比例，在工作日大约是百分之三十五，到了周末会上升到接近百分之五十。也就是说，周一到周五，Claude 更像一个白领工具，帮人写商务邮件、营销文案、幻灯片、报告。到了周六周日，它又变成生活顾问、情绪陪伴、健康咨询助手、投资讨论对象。

这个现象在高收入国家更明显。你可以想象一个典型场景：一个在大城市工作的产品经理，周三上午让 Claude 帮他改一封发给客户的邮件，下午让 Claude 梳理竞品分析，晚上可能还让 Claude 帮他总结会议纪要。到了周日，他不想再看工作材料了，于是问 Claude，下周去日本旅行怎么安排，最近睡不好该怎么办，或者某个 ETF 值不值得长期持有。这些需求都发生在同一个用户身上，但在不同时间，Claude 扮演的角色完全不同。

类似的周末模式也出现在 Claude Code 和第一方 API 流量里，不过它们的个人用途基线更低。也就是说，开发工具和 API 天然更工作化，但周末还是能看到一点转向个人项目的痕迹。报告进一步拆了 Claude Code 的请求集群，发现周末下降最多的是后端架构、API 调试、数据存储这类典型工作任务；周末上升最多的是 AI agent 设计、量化交易、游戏。这个观察非常贴近工程师生活。很多人工作日改公司服务，周末就折腾自己的副业、自动化脚本、交易策略、小游戏，或者研究怎么做一个 agent 产品。

报告还看到一个有趣现象：创业相关对话在周六和周日最高。它把创业活动定义得比较宽，包括创业想法、副业变现、融资、创作者变现、电商、商业模式、医疗健康业务、社会企业、活动业务等等。这说明周末可能给人留出了探索新事业的空间。可是求职申请相关活动反而会在周末下降，和其他工作任务一起减少。这也很合理。写简历、投岗位，心理上更像工作义务；想副业、想创业，则更像周末给自己开的脑洞窗口。

如果把时间粒度再缩到小时，Claude 的使用就更像一张生活作息图。报告说，人们在当地时间早上七点最常问新闻。商务通信，比如写邮件，在工作日曲线上升，大概上午十点到十一点有一个小峰值。菜谱请求在晚上六点特别集中，是平均水平的二点三倍。媒体推荐集中在晚上，人们快下班或吃完饭后想找电影、剧集、播客、书。睡眠建议则集中在黎明前几小时，尤其大约凌晨五点。这个画面很真实：凌晨五点还没睡着的人，不会打开 Claude 问后端架构怎么设计，他更可能问，为什么我总在这个时间醒来，怎么才能重新入睡。

报告还看了夜间和周末的工作请求，发现当人们在非传统工作时间使用 Claude 做工作时，这些任务更偏向高工资职业。比如营销经理、程序员这类高收入或高技能岗位，更可能在晚上和周末继续工作。相反，与低收入四分位职业相关的任务，比如电话营销、文书类工作，在夜间和周末占比下降。Anthropic 也做了稳健性检查，把计算机和数学相关任务拿掉后，高收入职业任务在夜间和周末上升的模式仍然存在。这一点很值得我们思考。AI 使用日志不只是反映工具偏好，也反映劳动市场结构。那些工作边界更模糊、责任更结果导向的人，更容易把 AI 带到晚上和周末。

还有一个特别典型的节奏，是税务截止日。报告覆盖了美国报税截止期。四月十五日是美国个人报税的一个关键日期。结果在四月十四日，税务相关会话是五月平均日的八倍，四月十五日仍然维持很高，四月十六日就急剧下降。这个现象像极了搜索引擎时代的关键词趋势，只不过现在人们不是只搜索报税问题，而是把自己的具体情况拿来问 Claude，让它解释表格、梳理扣除项、提醒风险。AI 从信息检索工具进一步变成了临时专业顾问。

第二个核心点，是 artifact，也就是用户从 Claude 会话中真正带走了什么。Anthropic 用分类器把 chat 和 Cowork 会话按产出分成三十多个类别。它发现，百分之九十三的 Claude conversations 都产生了某种 artifact。最常见的是解释，占百分之十七；文档和报告，占百分之十五；指导建议，占百分之十一。总体来看，解释和指导这类对话型输出，以及文档、演示文稿这类书面交付物，各占大约三分之一；代码和技术工作，比如应用、脚本，大约占六分之一。

这里要注意一个关键区别：一个产物是什么，不等于它用来干什么。同样是一份计划，可能是个人旅行计划，也可能是公司增长策略。同样是一段创意写作，可能是同人小说，也可能是短视频脚本。报告把 artifact 和用途分类结合起来看，分为工作、个人、课程学习。

有些 artifact 几乎总是个人用途。比如创意写作、指导建议、菜谱，超过百分之八十都被归为个人。即便同一类别内部，个人和工作用途也很不一样。个人创意写作里，常见的是同人、世界观设定、诗歌；那百分之十三工作相关的创意写作，则更多是短视频脚本、影视剧本、演讲稿。工作用途最明显的类别包括营销内容，工作相关比例大约百分之八十；博客或文章，约百分之八十一；数据库查询，约百分之八十二。

还有很多 artifact 同时服务个人和工作。比如计划或策略，百分之四十四工作相关，百分之四十九个人相关。个人计划常见是旅行路线、健身安排；工作计划则常见是创业策略、内容策略。翻译也是这样，工作和个人都很多。课程学习相关产物，则更集中在学术论文、教育材料、数学问题上，但这些类别也不是纯粹学习用途，有一部分会落在工作或个人。

反过来看也很有意思。工作会话最常产出文档和报告，占百分之二十，其次是解释、邮件草稿、分析和摘要。课程会话也类似，文档和报告占百分之二十一，解释占百分之二十，教育材料占百分之十一，学术论文占百分之六。个人会话就不一样，只有百分之六会产出文档，最常见的是解释，占百分之二十五，以及推荐，占百分之二十二。这个结果告诉我们，如果你在做企业 AI 产品，别只问模型能不能回答问题，要问用户最后想拿走什么。企业场景里，artifact 往往是报告、邮件、总结、分析、代码、演示文稿。个人场景里，很多时候人要的是理解、建议和选择。

第三个核心点，是算力成本和工作价值之间的关系。Anthropic 用 token 来衡量每段会话的计算成本。token 可以理解为模型处理和生成文本的基本单位，这里还包括 Claude 内部推理消耗的部分。报告把工作相关会话映射到通常执行这些任务的职业，再拿职业工资来比较。结果发现，一个职业对应的工资越高，相关会话的 token 中位数通常也越高。换句话说，更高价值的工作，往往需要更多模型计算。

报告举了一个例子。营销经理每小时工资大约八十美元，编辑大约三十七美元，营销经理相关任务工资差不多是编辑的两倍，而映射到营销经理任务的会话消耗 token 大约是编辑任务的二点五倍。当然这个关系很 noisy，也就是噪声很大，不是所有职业都符合。比如药剂师工资接近统计助理的三倍，但映射到药剂师任务的会话 token 只有统计助理的约二十分之一。这说明工资、任务复杂度、Claude 当前可承担的部分，并不是一条简单直线。

从 artifact 类型看，也能看到类似趋势。构建应用的会话消耗 token 超过中位会话的三倍，而一个典型解释只消耗中位会话的大约五分之一。报告估计，职业工资和 token 消耗之间的梯度，有百分之四十四可以由输出类型组合解释。高工资职业更可能产生计算密集型 artifact，比如应用、复杂文档、策略分析，而不是简单问答。

这个发现的经济含义很重要。报告发现，在映射到高工资职业的会话里，Claude 每轮产生的输出更多，大约是一点三四倍；用户参与也更多，轮次数量大约是一点五三倍；开启 extended thinking，也就是扩展思考的比例也略高，百分之三十四对百分之三十一。关键是，这些是一起增加的。Claude 产出更多，并不意味着用户更少参与。相反，在高价值任务里，人和 AI 都更忙。这让报告倾向于把这种模式理解为 labor augmenting，也就是增强劳动，而不是简单 labor displacing，也就是替代劳动。

如果你在公司里推动 AI 落地，这一点非常关键。很多管理者直觉上会问，用 AI 后人是不是可以少做点？但在高价值任务上，更常见的现实可能是，人会做更多判断、更频繁迭代、更快探索更多方案。AI 把产能天花板抬高了，但不是把人从流程里拿掉。比如一个资深工程师让 Claude Code 重构一个模块，他不一定按下回车就去喝咖啡。更可能是看模型理解是否正确，检查测试失败原因，要求它改方案，决定哪些改动可以合并。token 在烧，人脑也在工作。

第四个核心点，是自治程度，也就是 Claude 有多少自主判断空间。报告用一到五的量表衡量，从没有自治到极高自治。容易精确描述的任务，自治程度低，比如数学计算、翻译、问答。因为答案空间比较明确，你只要告诉它算什么、翻译什么、回答什么。高自治任务则需要在很多可能选择中做判断，比如做应用、做网站、做游戏、做演示文稿。这类工作历史上更难自动化，因为它不是只执行步骤，而是要持续判断什么方案更好。

报告比较了 chat 和 Cowork，与 Claude Code 的差异。结果在三十一个输出类型中的二十六个里，Claude Code 上 AI 自治程度更高。比如生成脚本和代码片段时，Claude Code 会话的自治评分平均比 chat 或 Cowork 高零点五三分。总体平均差异是零点三七分。这个差异有两个来源。大约三分之二来自同样任务在 Claude Code 里被委托得更多。报告用博客和文章举例：chat 和 Cowork 里生成博客的中位会话有十三轮来回，而 Claude Code 里生成博客的中位 session 只有一个人类 prompt。剩下三分之一来自不同界面的输出类型组合不同。

你可能会问，这是不是因为 Claude Code 更常用更强模型？报告也考虑了。Claude Code 中有百分之五十四由 Opus 服务，而 chat 和 Cowork 只有百分之十左右用 Opus。可是当比较同一个模型时，差异仍然存在。比如都用 Sonnet 的会话里，Claude Code 的自治程度仍然高零点二六分。这说明产品形态本身很重要，不只是底层模型强不强。一个界面如果天然支持读文件、改文件、跑命令、长任务，它就鼓励用户把整块任务交出去。一个聊天框如果主要是对话，它就鼓励用户一轮一轮澄清和控制。

报告还发现，用户委托最多的输出类型，往往也是消耗 token 最多的输出类型。在 artifact 层面，平均自治程度和 token 使用中位数一起上升，相关系数在 chat 和 Cowork 上是零点六八。这个数字不是因果证明，但直觉很清楚：越是需要 AI 自己规划、尝试、修改、判断的任务，越耗算力，也越接近真正的自动化工作流。

第五个核心点，是 Claude 的回答通常比用户提问高一个阅读层级。报告用分类器估计每段会话里，用户 prompt 和 Claude response 各自需要多少年教育才能理解。学术论文相关会话的平均 query 需要超过十六年教育，大约相当于本科水平，其中百分之十五达到博士或以上，也就是二十年以上教育水平。菜谱和指导建议这类会话则低很多，prompt 理解门槛低于十年教育。

总体上，输出阅读水平高的 artifact，输入 prompt 阅读水平也高，相关系数零点八七。但几乎每个类别里，Claude 的输出都比 prompt 高，平均高大约一年教育。差距最大的是用户描述要构建某物的场景，比如图像和图形高二点六年，游戏高一点九年，应用和网站高一点七年。原因可能有一部分是文体差异。用户 prompt 往往短、口语、含糊，而 Claude 回答倾向于结构化、正式、完整。面向受众的写作，比如博客、学术论文、邮件，差距就接近零，因为用户通常会提供同一文体的草稿或素材。

这个发现对提示词设计有启发。很多时候，用户不是缺知识，而是把需求说得很粗糙。AI 会把粗糙需求翻译成更正式、更高层级的表达。但这也有风险。如果你要面向普通用户写说明，Claude 默认可能写得过于正式、过于高级。你需要明确告诉它目标读者是谁，用什么阅读水平、什么语气、什么场景。否则它生成的内容看起来专业，却可能不好读。

接下来我们进入调查部分。前两部分主要讲日志，第三部分问人。Anthropic 在二〇二五年十二月曾用 Anthropic Interviewer 采访过八万一千名 Claude 用户，发现大家报告了很大的生产力提升，同时也担心被替代，尤其是早期职业阶段的人，以及那些 Claude 已经做了很多工作的职业。二〇二六年四月，他们启动了新的经济指数调查，并把五月中到六月初的使用数据以隐私保护方式关联起来。每个人随机抽最多二十个 session，覆盖 Claude 的不同使用界面。少于五个 session 的人被排除，以降低噪声。最终关联样本大约九千七百人。

首先要记住，样本不代表普通人。计算机和数学职业严重过度代表，占受访者大约百分之三十，而在美国就业中只有百分之四。管理岗位也过度代表，占百分之二十三，而美国就业份额大约百分之七。交通运输、餐饮、建筑等体力类职业则明显不足。也就是说，这份调查非常适合理解 Claude 重度知识工作者怎么看 AI，但不能直接推广成全社会民意。

调查问了一个核心问题：你觉得 AI 今天能完全独立完成你工作任务中的多少？十二个月后又能完成多少？选项是从几乎没有到几乎全部的五个区间。接近六成受访者选择了更高的明年区间。超过三分之一的人预计，明年 AI 能做他们大部分或几乎全部工作任务。

这里有一个很微妙的结果。受访者报告的当前 AI 能力，和 Anthropic 之前构建的 observed exposure，也就是实际观察到 Claude 正在做的职业任务比例，呈正相关；也和 theoretical exposure，也就是理论上大语言模型能做的任务比例，呈正相关。说明人的感受不是乱来的，大体和外部指标对得上。但是，当问未来十二个月提升多少时，不同职业的预期增量差不多。报告说，一名软件工程师和一名施工经理，都大致预期自己的职业内 AI 能力会有类似幅度的进步。这像一种 rising tide，涨潮效应。大家觉得 AI 会全面变强，而不是只在某些职业里变强。

报告还发现，受访者报告的 AI 当前能力，系统性高于 observed exposure。可能原因是，调查触达的是更常用 AI 的人，而且一个职业里的每个人实际做的任务不同。另一个方向，theoretical exposure 又通常高于 reported exposure，因为理论可做不等于今天实际好用。这个三角关系很好理解。理论指标像天花板，日志指标像已经发生的地面事实，用户自评则在中间，带着自己的工作上下文和使用经验。

不同国家和经验年限也影响感知。报告说，人均 GDP 越高的国家，受访者认为 AI 目前能做的任务比例越低，高收入国家大约低十个百分点。这个结果乍看反直觉，因为发达经济体职业通常被认为 AI 暴露更高。但报告提出一种解释：在低收入国家，AI 可能替代了工人日常任务中更大比例；而在高收入国家，AI 更可能和人的技能、基础设施、组织流程互补。也就是说，同样是 AI，进入不同经济结构后，像替代还是像增强，可能不一样。

工作经验越多的人，也越不认为 AI 能做自己大量工作。十五年以上经验的人，对 AI 当前可完成任务比例的估计，比第一年工作的人低大约十个百分点。后续开放问题显示，资深工作者常说 AI 缺少判断、上下文意识、情境推理，以及建立信任、管理人际关系的能力。我个人认为，这一点对工程师尤其重要。初级任务很多是显性规则，比如写一个接口、修一个语法错误、整理一个文档。资深任务很多是隐性上下文，比如这个改动会不会伤害团队协作，客户真实需求是不是另一个方向，技术债现在该不该还。AI 在前者更容易显得强，在后者更容易露出短板。

报告还比较了用户和 Claude 的协作模式。它把 automated，也就是自动化模式，定义为 Claude 被要求在很少用户输入下完成任务。具体包括 directive，直接指派任务，比如翻译这份文档；以及 feedback loop，反馈循环，比如先改这封邮件，再让它更随意一点。和自动化相对的是更增强式的模式，比如学习、验证、共同迭代。结果显示，自动化使用占比越高的人，越认为 AI 今天和十二个月后能做更多自己的工作任务。原因可能有两个：一是敢把任务交出去的人亲眼看到 AI 能完成什么；二是本来相信 AI 强的人更愿意交出去。报告不能完全区分因果。

更有意思的是，自动化使用更多的人，并没有更悲观。相反，在工资、工作安全、找新工作能力、工作意义、自主性、人际互动这六个维度上，自动化会话占比越高的人，对未来一年 AI 影响越乐观。最大的正向差异出现在对未来工资和找工作能力的预期上。报告也检查了一个可能的选择偏差：是不是 AI 爱好者既更愿意自动化，也更乐观？他们用 Claude 使用年限作为热情程度的代理控制后，结果没有明显变化。当然这不等于因果证明，但至少说明关系很稳。

为什么会这样？报告认为，一个可能原因是，自动化使用者今天已经体验到更多收益。大多数人报告 AI 提升了工作速度、工作范围和工作质量，比例分别是百分之八十六、百分之八十二、百分之六十九。还有百分之二十七的人说，因为 AI，他们节省了原本要购买外部服务的成本。多数人还报告自己学得更多，比例是百分之六十八；觉得 AI 让自己的技能更有市场价值，比例是百分之五十七。随着自动化会话占比上升，认为自己技能市场价值提升的人更多，而报告学到更多的人大致持平。

这挑战了一个常见担忧。很多人说，把整个任务交给 AI，会不会导致人不思考、不学习、技能退化？报告没有看到重度委托者自评学习减少。但它也非常谨慎地说，这些都是自我评估，技能可能在变得更值钱的同时也发生某些退化，所以数据不能排除技能侵蚀。这里我个人认为，对工程师最现实的策略不是拒绝自动化，而是设计有意识的学习回路。比如你可以让 Claude Code 先做方案，但要求它解释关键决策；让它修测试，但你必须 review diff；让它生成迁移脚本，但你要写验收标准。这样自动化产出和人类能力增长不是二选一。

报告还问了工作变化和失业风险。超过三分之一受访者认为，未来十二个月里，自己、同级、初级同事或高级同事的职责很可能显著改变。百分之十认为自己失业 likely 或 very likely，也就是可能或非常可能。这个数字略低于美国年化非自愿离职事件发生率，但考虑到样本偏知识工作者和稳定就业者，仍可能代表较高的主观风险。那些认为自己可能失业的人中，有百分之三十八在开放回答中把预测归因于 AI。受访者普遍更担心别人失业，而不是自己，尤其担心初级同事。超过三分之一的人认为，初级同事明年失业概率超过百分之六十。这个结果和很多团队体感相符：AI 对初级岗位的冲击，至少在心理预期上非常明显。

报告还提到性别差异。在关联样本中，女性只占百分之十二，而且女性使用 Claude 的方式和男性不同。即使控制职业差异，女性也略少把 Claude 用于工作，在 Claude Code 中的 session 份额低六点三个百分点，自动化占比低七点三个百分点。相对地，女性更倾向于迭代式使用，在 chat 中活跃时间更长，显示出更协作式的参与。报告没有把原因展开，这里需要谨慎，不要过度解释。但对产品设计者来说，它提醒我们，agentic 工具的采用方式可能存在群体差异。一个只用自动化率衡量成功的团队，可能低估了协作式使用的价值，也可能忽视某些用户群体的需求。

调查最后问了一个很开放的问题：大胆想象，十年后，一个由 AI 塑造的经济，你希望它是什么样？最常见主题是工作增强。超过一半受访者希望能和 AI 协作，继续做有意义的工作，希望自己的职业仍然重要，或者希望新产业出现、新机会出现。与此同时，略高于一半的人也希望 AI 自动化，尤其自动化工作里乏味、重复、耗人的部分，让自己有更多自由时间和工作之外的意义。第三大主题是 shared prosperity，也就是共同繁荣，约三分之一受访者希望 AI 带来的经济收益被广泛分享。

这部分让我觉得这篇报告不只是冷冰冰的数据。很多真实用户的期待并不是简单地说，让 AI 替我上班，或者让 AI 别碰我的工作。他们想要的是更复杂的平衡：保留有意义的部分，去掉折磨人的部分；让技能变得更有价值，而不是被清零；让生产率收益不要只流向少数公司或少数人。这也许是未来几年 AI 产品、组织管理和公共政策都绕不开的核心问题。

那这篇报告对我们这些工程师和 AI 从业者有什么实际启发？我觉得至少有几条可以马上带走。

第一，不要只按功能分类 AI 使用，要按时间和场景分类。工作日和周末、上午和晚上、税务截止日前后，用户需求完全不一样。如果你在做内部 AI 平台，可以看看公司里的使用节奏。早上是不是更多总结新闻和同步信息？上午是不是更多邮件和会议材料？下午是不是更多代码和数据分析？晚上是不是更多个人学习和探索？这些节奏会影响你的缓存、配额、成本控制、功能推荐，甚至影响你什么时候推送什么模板。

第二，产品设计要围绕 artifact，而不是围绕聊天。很多 AI 应用还停留在一个输入框加一个回答框。但报告提醒我们，用户真正要拿走的是可交付物。对工程团队来说，这意味着你要把 artifact 类型定义清楚。比如代码场景，最终产物是 pull request、测试报告、迁移说明，还是一段脚本？销售场景，最终产物是客户邮件、通话摘要、报价草案，还是 CRM 更新？法务场景，最终产物是风险清单、合同修订建议，还是审批意见？当你围绕 artifact 设计，评估指标也会更清楚：不是模型回答得像不像，而是产物能不能被使用、能不能被审查、能不能进入流程。

第三，自动化程度要分层，不要一上来就追求无人驾驶。报告里低自治任务和高自治任务差别很大。翻译、计算、问答可以较低自治；网站、应用、游戏、演示文稿需要高自治。如果你在公司落地 agent，不妨给任务做一个自治评分。这个任务的输入是否足够明确？失败成本多高？有没有自动化验收？用户是否能快速 review？依赖外部系统吗？如果一个任务高自治、高失败成本、低可验证性，那就不适合直接全自动。反过来，如果高自治但有强测试，比如代码修改后能跑单测、集成测试、静态检查，那就可以逐步放权。

第四，要把 token 成本当作经济信号，而不是单纯技术成本。报告显示，高价值任务往往消耗更多 token。很多公司做 AI 成本治理时，容易简单按部门限额，或者按人头限额。但更好的方式是看产出价值。一个简单解释耗费少，价值也可能低；一个复杂代码重构耗费高，但如果节省资深工程师两小时，可能非常划算。你需要把 token、任务类型、artifact、人工 review 时间、最终业务价值连起来。否则很容易出现两种误判：要么为了省 token 阻止高价值任务，要么放任低价值闲聊消耗预算。

第五，要重视人机共同参与，而不是把人的参与看成自动化失败。报告里高工资职业任务中，Claude 输出更多，用户轮次也更多。这说明最有价值的场景可能不是一键完成，而是高强度协作。比如架构设计、复杂调试、策略规划，AI 提供速度和备选方案，人提供上下文、目标函数、风险判断。管理层如果只问这个 agent 能不能把人替掉，可能会错过真正的生产力提升。更好的问题是，它能不能让一个工程师同时探索三个方案？能不能让一次设计评审从空泛讨论变成基于具体草案的讨论？能不能让 junior 更快接近 senior 的思考框架，同时让 senior 有更多时间做关键判断？

第六，团队要明确保留学习回路。自动化使用者更乐观，并不代表没有技能退化风险。工程团队可以建立一些实践，比如要求 AI 生成的代码必须附带设计说明；要求关键变更由人写验收标准；要求复杂 bug 修复后做一次事后解释；要求 agent 输出不仅有结果，还有它尝试过的路径和失败原因。这样，AI 不只是替你干活，也成为一个外显思考过程的工具。

第七，不要忽视初级岗位的变化。报告里大家尤其担心 junior colleague，也就是初级同事的失业风险。对工程组织来说，这不是一句空话。如果大量入门任务被 AI 承担，初级工程师靠什么成长？过去他们通过修小 bug、写小功能、改文档、跑测试积累系统感。现在这些都可能被 Claude Code 做掉。公司需要重新设计 apprenticeship，也就是学徒式成长路径。比如让新人负责定义任务、写测试、review AI diff、维护评估集、做线上问题复盘，而不是只把他们从简单执行中移除。否则几年后可能出现一个断层：AI 很强，senior 很忙，但缺少真正成长起来的中坚工程师。

第八，产品和组织都要考虑不同协作风格。报告里的性别差异虽然不能简单推广，但它提示我们，不同人对 AI 的信任建立方式不同。有的人喜欢一句话交给 agent，有的人喜欢多轮迭代、逐步控制。一个好的 AI 产品应该同时支持这两种模式。比如既有一键执行的高自治工作流，也有可暂停、可解释、可编辑、可比较版本的协作界面。对企业内部推广来说，也不要只奖励自动化率。有人用 Claude 作为学习伙伴、审稿伙伴、验证伙伴，也可能产生很大价值。

最后，我们回到这篇报告本身。Anthropic 在讨论部分说，AI 正在通过越来越多界面快速扩散到经济中，输出也越来越智能。早期聊天界面比较简单，没有网页搜索、工具调用、artifact 或其他能力。现在 Claude 可以通过 Claude Code 和 Cowork 自主运行数小时。随着形式变化，用户群体也在变化。早期采用者高度技术化，而最新用户开始把 Claude 用到市场工资较低的任务上。

它也承认，准确分类 Claude 做的工作会一直是移动靶。随着 AI 能力提高，AI 之间可能越来越多地互相交互和交换信息，甚至以人类和简单分类器难以理解的方式协作。最终，Claude 对经济的影响不只会体现在使用日志里，也会体现在就业、生产率这些宏观经济指标中。但在最早阶段，AI 影响往往会先出现在它做最多工作的地方。所以持续观察这些变化中的使用模式，仍然是理解 AI 经济影响的重要窗口。

我个人认为，这篇报告最大的价值，是把 AI 从抽象能力拉回真实生活。它告诉我们，AI 不是均匀地降临到世界上，而是沿着人的作息、职业、收入、产品界面、任务产物、组织流程一点点渗透。早上七点的新闻、晚上六点的菜谱、凌晨五点的睡眠建议、报税日前夜的焦虑、周末的创业想法、工作日的邮件和代码，这些看似琐碎的节奏，合在一起才是 AI 进入经济的真实样子。

如果用一句话总结，Claude 的使用正在从聊天走向交付，从回答走向执行，从单次问答走向长时间代理任务。而对我们来说，真正的问题不再是要不要用 AI，而是怎么设计人与 AI 的分工，让它既能自动化繁琐工作，又不抽空人的判断、学习和职业意义。这就是这篇 Cadences 报告最值得我们带走的地方。

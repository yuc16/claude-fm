## Script

今天我们来聊一篇 Anthropic 发布的研究报告。原标题是"Anthropic Economic Index report: Uneven geographic and enterprise AI adoption"，我来翻译一下，大意是"Anthropic 经济指数报告：不均衡的地理分布与企业 AI 采用"。这篇报告发表于二零二五年九月十五日。

先跟你说一声时效性的问题。今天是二零二六年六月十五日，这篇报告距今已经整整九个月了。在 AI 领域，九个月是相当长的时间，很多具体数字可能已经有了变化。但我认为这篇报告依然非常值得听，原因是它记录的是一个关键时间节点——二零二五年下半年，正是 Claude Sonnet 四成为主力模型、各家大模型能力出现明显跃升、企业对 AI 的态度从观望转向真正开始规模化部署的时期。这份报告就像那个时间节点上的一张高清快照：告诉我们人类究竟在用 AI 做什么、在哪里用、用多少、怎么用。这些早期扩散的规律，就像我们今天去研究十年前智能手机早期应用生态的增长数据，具体的月活数早就变了，但那些关于用户行为和市场渗透的结构性规律，对理解当下依然有价值。

好，进正题。

这份报告要回答的核心问题是：AI 是怎么在全球范围内扩散的？谁在用，在哪用，用来做什么，怎么用？这些问题听起来很学术，但答案决定了一个非常现实的问题——AI 的生产力红利最终会流向谁。

先从一个你可能听说过、但可能没仔细想过的数据开始。Anthropic 引用了一份 Gallup 的调查结果：在美国，有百分之四十的员工表示自己在工作中使用 AI，而两年前这个数字只有百分之二十。翻倍，两年时间。

你可能觉得这速度没什么特别的，新东西出来大家不都抢着用吗？但报告给了一个历史对比，让这个速度的含义变得非常清晰。电力普及到美国农村花了三十年以上——从城市通了电，到农村家庭普遍用上电，三十年。第一台量产个人电脑是一九八一年推出的，整整二十年后个人电脑才进入美国大多数家庭。互联网历来被认为是传播最快的技术之一，但它也花了将近五年时间，才达到 AI 用两年就达到的采用率。

为什么这次不一样？报告给出了三个原因。第一，AI 可以直接部署在现有的数字基础设施上，不需要重新建工厂、铺管道、修电线，只要有网就能用。第二，使用门槛极低，只需要打字或者说话，零学习曲线，不像学电脑还要学鼠标键盘，不像学开车还要考驾照。第三，AI 本身的能力在快速进化，进化本身又反过来吸引更多人来用，形成正反馈。

但扩散快不等于扩散均匀。报告的核心主张是：AI 的早期采用高度集中——集中在少数地区，也集中在少数类型的任务上。报告说这是历史上所有重大技术在早期都会呈现的模式，只是 AI 把这个时间尺度压缩了。这份报告就是来记录和分析这种集中模式的。

整份报告分三个章节：第一章讲 Claude.ai 在过去八个月里的用法变化；第二章是这次最大的亮点，第一次发布了全球一百五十多个国家和美国所有州的地理使用数据；第三章讲企业通过 API 接入 Claude 的模式，也是首次发布。我们一个一个来拆解。

先说第一章，关于使用内容和模式的时间变化。

Anthropic 收集了三个时间点的数据，他们自己管这三组叫 V1、V2、V3。V1 是二零二四年十二月到二零二五年一月，V2 是二零二五年二月三月，V3 是二零二五年八月。横跨八个月，三张快照，让我们看到了使用方式的演变。

最稳定的结论是：编程任务依然是第一大用途，占总用量的百分之三十六，这个份额在三个时间点里基本没怎么变。但是边上的变化很有意思。教育类任务从 V1 的百分之九点三增长到了 V3 的百分之十二点四；科学类任务从百分之六点三增长到了百分之七点二。与此同时，商业和金融操作类任务从百分之六降到了百分之三，管理类从百分之五降到了百分之三。

这个结构性变化意味着什么？报告的解释是，Claude 在推理、解释、知识综合方面的能力提升，让它在知识密集型任务——教学、科学研究、学术写作——里变得特别有用。相对而言，传统商业事务类任务可能更多依赖固定流程和规范化模板，对 Claude 的开放式推理能力需求没那么高，所以增速没有教育科学类快，份额被相对压缩了。

有几个数字的变化可以直接和产品功能对应上。网络搜索相关任务的占比从零点零三跳到了零点四九，几乎是十六倍，时间节点正好对应 Anthropic 在二零二五年三月推出联网搜索功能。互联网研究任务从零点零零三增长到了零点二七，对应的是四月份推出的 Research 模式，也就是让 Claude 能自主搜索多个来源来完成深度研究任务的功能。这些数据几乎是实时记录了用户如何把新功能融入自己的工作流。

还有一个值得关注的代码相关变化：创建新代码的任务占比超过翻倍，从百分之四点一增长到了百分之八点六；与此同时，调试和错误修复的占比从百分之十六点一下降到了百分之十三点三，一增一减，净差是七点多个百分点向创建方向移动。报告的猜测是，Claude 写出来的代码 bug 少了，用户自然不需要花那么多时间去修，可以把精力放在写新功能上。

用法模式上最值得关注的变化是：越来越多的用户直接把任务交给 Claude 完成，而不是一起来回迭代。

报告把人机协作方式分成两大类。一类叫自动化，用户把任务整个交给 Claude 做，分两个子模式：指令式，就是一次性给 Claude 一个任务让它做完，不需要来回；反馈循环，用户会看结果然后继续给指令，但主导权在 Claude 这边。另一类叫增强，是更协作的模式：一起学习、来回迭代、让 Claude 帮你检查工作成果。

数据上，指令式对话的占比从 V1 的百分之二十七跳到了 V3 的百分之三十九，短短八个月增加了十二个百分点。报告说这是它们第一次观察到自动化用量超过增强用量。

这个数字背后有两种解释，而两种解释对劳动力市场的含义是截然不同的。第一种解释是模型变强了——Claude 能一次性给出好结果，用户不需要来回修改，所以自然而然就少了来回。第二种解释是用户在学习——随着经验积累，他们越来越善于一次性给出精准的指令，越来越信任 AI 的输出。

如果是第一种，模型自动化了更多任务，某些职位被取代的风险上升。如果是第二种，善于使用 AI 的人效率大幅提升，这类人反而变得更值钱。报告明确承认这两种解释今天还无法区分，但指出这是未来研究最需要回答的问题之一。

好，进入第二章，也是这份报告最有分量的新内容：全球和美国各州的地理使用分布。

Anthropic 分析了二零二五年八月随机抽取的一百万条 Claude.ai 对话，用 IP 地址判断地理位置，覆盖了一百五十多个国家和美国所有州。这是 Anthropic 第一次公开这个层面的数据。

为了能在不同规模的国家之间做有意义的比较，报告引入了一个新指标，叫做 Anthropic AI 使用指数，英文缩写 AUI。计算方式很直接：一个地区的 Claude 使用量份额，除以这个地区劳动年龄人口的份额。这个值大于一说明这个地区用 Claude 的频率高于按人口比例应有的水平，小于一说明偏低。

你可以这么理解这个指标：如果一个国家占全球劳动年龄人口的百分之五，但在我们抽到的 Claude 对话里占了百分之十，那它的 AUI 就是二，意味着这个国家的人用 Claude 的频率是全球平均的两倍。

具体数字来了。全球前五：以色列 AUI 是七，也就是说以色列人用 Claude 的频率是全球平均水平的七倍；新加坡是四点五七，澳大利亚是四点一，新西兰是四点零五，韩国是三点七三。

紧接着是一些你可能熟悉的主要国家：美国是三点六二，加拿大是两点九一，英国是两点六七。欧洲主要国家稍低：法国一点九四，日本一点八六，德国一点八四。

另一头：印度是零点二七，印度尼西亚是零点三六，尼日利亚是零点二。也就是说，相对于人口，这些国家用 Claude 的频率只有全球平均的五分之一甚至更低。

这种巨大差距背后是什么？报告给出了几个层次的解释。数字基础设施，高 AUI 国家通常有更好的网络连接和云计算条件。经济结构，发达国家有更高比例的知识工作者，而 Claude 最擅长的偏偏就是这类任务，比如写报告、写代码、做分析；低收入国家有更大比例的制造业和农业就业，这些任务目前和 AI 的匹配度低很多。监管环境，有些国家政府积极推动 AI 应用，有些则限制更多。还有对硅谷和 AI 研究社区的接近程度，以及公众对 AI 的信任态度。

统计上，报告发现人均 GDP 和 Claude 人均使用量之间有很强的正相关：人均 GDP 每增加百分之一，Claude 人均使用量大约增加零点七个百分点。这个关系跟历史上电力、电话、互联网等技术的早期扩散规律高度吻合。

在美国境内，加利福尼亚州在绝对用量上排第一，占全美百分之二十五点三。但按人口调整之后，画面很不一样。华盛顿特区以三点八二的 AUI 领先全美，犹他州以三点七八紧随其后，反而超过了加州的两点一三。纽约是一点五八，弗吉尼亚是一点五七。

DC 排第一在我看来其实很好理解。政府机构、智库、律师事务所、咨询公司在那里密集聚集，知识工作者浓度极高，而且公文起草、政策分析、职业材料这类文字密集的任务需求很大。数据印证了这个猜测：帮写求职申请材料在 DC 的比例是全美平均的一点八四倍。

各地区的使用内容也有明显的本地化特征，和当地经济结构高度对应。加州用户在 IT 相关任务和数字营销上的使用比例高于平均，与当地科技产业吻合。佛罗里达州有比较多的商业建议和健身健康类需求，可能和它作为金融中心以及温暖气候吸引健身文化有关。

跨国比较里，印度用户超过一半的用途是写代码，而全球平均只有大约三分之一是代码。巴西有偏高的翻译和法律服务需求，背后是巴西司法系统是全球早期采用 AI 的法律体系之一。越南用户集中在软件开发和教育。这些差异非常直观地反映了各地经济结构对 AI 使用方式的塑造。

为什么不管在哪个国家，编程都是最大的单一用途？报告给出了三个原因：Claude 本身是非常强的代码模型；程序员社群接受新工具的速度历来很快，而且这种使用习惯会在开发者网络里迅速传播；另外，程序员个人采用 Claude 通常不需要经过复杂的内部审批，不像医疗、法律这些受到严格监管的行业，个人使用门槛极低。

报告还有一个跨国的反直觉发现，我觉得值得单独说一说。在控制了各国任务类型差异之后，AI 采用率越低的国家，用户越倾向于把任务直接委托给 Claude 完成，也就是自动化；而采用率越高的国家，用户反而更倾向于和 Claude 协作迭代，也就是增强。你可能以为越懂 AI 的地区应该越会用自动化，实际数据恰好相反。报告猜测这可能和文化因素或者经济发展阶段有关，但承认还需要更多研究。

报告在地理章节的结尾明确表达了对全球不平等的担忧。历史上每一次重大技术革命——电气化、内燃机、现代农业——都同时伴随着全球生产力差距的扩大。如果 AI 的生产力收益主要集中在已经富裕的地区，那它可能加剧而不是缩小全球不平等，逆转近几十年来全球经济收敛的趋势。这不是假设性的担忧，而是需要政策层面认真对待的真实风险。

好，现在来说第三章，也是最有企业参考价值的部分：Anthropic 第一次公开分析了他们的企业 API 客户数据。

先说一下 API 和 Claude.ai 的本质区别。Claude.ai 是面向个人的产品，你打开网页或者 app，直接跟 Claude 对话，按月付订阅费。API 是企业用代码调用 Claude，把它嵌进自己的系统、应用、业务流水线里，按实际使用量付费，也就是按处理的文字量付费。数据来自二零二五年八月随机抽取的一百万条企业 API 对话记录，来自大约一半的 Anthropic 企业 API 客户群。

第一个发现：企业用 Claude 和个人用户有相似之处，但更加专注。两者都以编程任务为第一大用途，但企业 API 里代码的占比比 Claude.ai 高出八个百分点以上。用职业分类来统计，接近一半的 API 流量对应计算机和数学类任务。另外大约百分之十是办公和行政任务，这类任务非常适合自动化。

反过来，Claude.ai 上占比比较高的教育任务，在 API 里占比从百分之十二点三降到了百分之三点六；艺术娱乐类从百分之八点二降到了百分之五点二。企业不太会通过 API 来用 AI 学习或者创作文艺内容，这很好理解。

第二个发现，也是最显眼的数字：企业 API 对话里有百分之七十七是自动化模式，而 Claude.ai 用户只有大约百分之五十。如果按具体经济任务的维度来看，差距更大：API 数据里百分之九十七的任务是自动化主导，Claude.ai 上只有百分之四十七。

这非常直观：企业通过 API 调用 Claude，通常是把它嵌进一条处理流水线，请求进来，Claude 处理，结果输出，整个过程没有人在中间一句一句去跟 Claude 聊天。这和我们个人使用 Claude 的体验有本质的不同。

第三个发现可能让你有点意外：价格对企业选择用 AI 做什么任务的影响非常小。

API 是按 token 计费的，也就是按处理的文字量付费，不同任务的成本差异相当大。计算机类任务的平均成本比销售类任务高百分之五十以上，但计算机类任务的使用量反而更多。整体来看，成本越高的任务，使用量越大——两者呈正相关。

报告做了控制其他变量的统计分析：成本每提高百分之一，使用量大约减少零点二九个百分点。换算一下，如果把某个任务的价格压低百分之十，使用量只会大约增加百分之三。价格弹性极低。这说明企业在决定用 AI 做什么时，主要看的是 AI 能不能把这件事做好、自动化这件事能带来多大经济价值，而不是这件事要花多少钱。

对于在做 AI 产品定价的人来说，这个发现很有参考价值：在企业市场的早期，能力比价格重要得多。把产品做强，比把价格压低更能撬动企业用量。

第四个发现，我个人认为是整个报告里最有实操价值的：上下文信息的获取，而不是模型能力本身，可能是企业规模化部署 AI 的真正瓶颈。

报告分析了一个很有意思的关系。对于 API 里的每一类任务，Claude 的输出越长，也就是任务越复杂，企业给 Claude 提供的输入也越长。统计上，这个关系的斜率是零点三八，就是说输入长度增加百分之一，输出大约增加零点三八个百分点，存在明显的边际收益递减。

这个关系本身不难理解，复杂的任务当然需要更多背景信息。但报告想指出的是一个现实困境：很多复杂任务所需要的上下文，根本就不是数字化的、集中存放的、Claude 能方便读取的形式。

举个例子来说。如果你想让 Claude 重构某个代码模块，可以把整个代码库丢给它。这个上下文是集中的、结构化的，技术上给 Claude 提供并不难。但如果你想让 Claude 为一个大客户制定销售策略，它需要的信息可能分散在客户关系管理系统里、销售人员的历史邮件里、会议纪要里、甚至只存在于老销售脑袋里、从没写下来的行业经验。这些信息怎么给 Claude？这不只是一个技术问题，而是一个组织问题和数据治理问题。

报告的结论是：对于很多复杂的业务任务，企业要想真正大规模部署 AI，可能首先需要做一次信息基础设施的升级——把分散在组织各处的知识集中起来、数字化、结构化，让 AI 能访问到。这个投入是真实的，不小，而且不是今天大多数企业已经在做的事。报告把这个问题称为"高影响场景部署的潜在瓶颈"，我觉得这个判断很准确。

好，聊完报告的核心内容，来说说对我们这些在一线工作的软件工程师和 AI 从业者意味着什么。

第一点：如果你是开发者，你处于全球 AI 使用曲线的最前沿。报告里几乎所有国家的数据都显示，编程是 Claude 使用量一致领先的单一类别。这意味着最好的 AI 辅助编程工具、最多的 prompt 工程经验、最活跃的社群讨论，都在围绕你的核心技能展开。这是一个先发优势，但也意味着这个领域的竞争正在加速。能够一次性给出清晰指令、得到高质量输出的人，和需要来回十几轮才能让 Claude 明白自己要什么的人，效率差距会越来越大。

第二点：关于企业部署 AI，第三章的上下文瓶颈分析非常值得认真对待。在你规划用 Claude 自动化某个业务流程之前，先问三个问题：Claude 需要什么信息才能完成这个任务？这些信息现在在哪里，是什么格式？如何让 Claude 能高效地访问到它们？这三个问题的答案，很可能比你选哪个模型、用哪个平台都更重要。很多企业 AI 项目失败不是因为模型不够强，而是因为把一个信息分散、流程混乱的业务流程直接喂给了 AI，然后发现效果很差，结论是 AI 没用——实际上根本原因是信息准备不到位。

第三点：这份报告关于地理不平等的担忧，对做 AI 产品的人也有一层值得思考的含义。我们今天在做的产品、在做的决策，实实在在地塑造着这个技术最终的扩散形态。如何设计产品让 AI 的好处更广泛地覆盖不同地区和人群，不只是政策制定者需要考虑的问题，也是产品设计和商业模式需要思考的问题。

最后做个总结。这份报告的核心观点可以用几句话概括：AI 扩散的速度前所未有，但形态高度集中——集中在富裕国家和高教育水平地区，集中在编程这类门槛低、模型能力强的任务，集中在早期采用企业的自动化场景。企业部署 AI 的核心驱动力是能力和经济价值，价格弹性出奇地低。AI 真正进入复杂业务场景的最大瓶颈，不是模型不够强，而是企业能不能把正确的上下文信息给到模型。

我个人对这份报告最大的感受是：它的价值不在于给出了定论，而在于它提出了正确的问题——并且用真实数据让这些问题变得清晰可见。AI 的好处最终会流向谁？会帮助缩小全球不平等还是加剧它？在企业大规模自动化推进的过程中，哪些工作者会获益，哪些会受损？这些问题今天都还没有答案。但现在正在发生的这段历史，会是回答这些问题最重要的原始材料。Anthropic 把这些数据开源出来，让所有人都能拿去研究，我觉得这是这份报告之外另一个非常有价值的动作。

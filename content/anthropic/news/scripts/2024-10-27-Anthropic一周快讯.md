大家好,欢迎收听 Claude FM,这里是 Anthropic 一周快讯。本期覆盖的是二零二四年十月二十七号到十一月二号这一周,一共两条快讯,我们按时间顺序来讲。

第一条,十月二十九号,新版 Claude 三点五 Sonnet 开始接入 GitHub Copilot,开发者在 Visual Studio Code 和 GitHub 网站上就可以直接选用 Claude 来写代码,这次合作一下子把 Claude 的编程能力推到了一亿多名开发者面前。这条值得多说两句,因为它不只是一次简单的渠道上架。Anthropic 强调,这版升级后的 Claude 三点五 Sonnet,在 SWE bench Verified 这个基准上的表现超过了所有公开可用的模型,这个基准衡量的是模型解决真实 GitHub issue 的能力,也就是说不是做做选择题,而是真的能不能修好一个实际的代码仓库问题。另外它在 HumanEval 这个考察用自然语言描述生成正确 Python 函数的基准上,拿到了百分之九十三点七的分数,是同类模型里的最高分。结合到 GitHub Copilot 这个场景里,开发者可以用它来根据需求描述直接生成符合项目代码风格的可用代码,可以高亮一段有问题的代码让它帮忙调试和解释报错,也可以选中一个函数自动生成对应的测试用例。这次集成会在接下来几周向所有 GitHub Copilot Chat 用户和企业账号逐步开放公测,我个人觉得这是 Claude 在开发者工具生态里渗透率的一次明显跃升,值得后续持续关注它在 Copilot 用户里的实际使用反馈。

接下来,十月三十一号,Anthropic 发了一篇态度鲜明的政策长文,核心观点是呼吁各国政府在未来十八个月内尽快采取行动,出台针对性的 AI 监管措施,文章原话是说主动防范风险的窗口正在快速关闭。这条也值得展开讲讲,因为它体现的是 Anthropic 一贯的政策立场,既不是放任不管,也不是一刀切式的严管,而是主张精准、窄范围的定向监管,目的是在享受 AI 带来的科学进步、医疗突破和经济增长这些好处的同时,把灾难性风险控制住。文章里提到,过去一年里 AI 系统在数学、研究生水平的推理、编程等多个能力上都有显著提升,而公司内部还有更多尚未公开的系统和研究结果在持续推进。Anthropic 的担忧是,这些能力进步本身是中性的,但同样的能力既可能被用在网络安全、生物安全等领域里造成破坏性应用,也可能带来 AI 系统自身行为失控的风险。文章警告说,如果各国政府现在拖延,将来很可能被迫匆忙出台一些设计粗糙、临时拼凑的监管,既拖累了 AI 行业的正常发展,又没能真正防住风险,这是最糟糕的结果。这篇文章某种程度上也是 Anthropic 在向各国监管机构喊话,把自己定位成希望与政策制定者合作、推动负责任监管落地的那一方。

整体来看,这一周一边是 Claude 在产品和开发者生态上的稳步扩张,一边是 Anthropic 在政策层面持续发声,我个人觉得后面这篇监管长文更值得花时间细读,毕竟它关系到整个行业未来几年的监管走向。本期快讯就到这里,我们下期再见。

大家好，欢迎收听 Claude FM。这里是 Anthropic 一周快讯，带你用几分钟回顾 Anthropic 官方这一周对外宣布和发生的事情。

本期覆盖的是七月五号到七月十一号这一周。本周一共有五条快讯，最值得关注的是两件事：一个是 Claude Code 被用于政府级网络安全修复，另一个是 Claude 开始进入更偏工业现场的物理 AI 场景。

先看第一条，七月六号，Anthropic 分享了加拿大阿尔伯塔省政府使用 Claude 的案例。

从二零二五年开始，阿尔伯塔省政府就在用 Claude Code，配合 Opus 和 Sonnet 模型，来审查政府系统里的代码，发现网络安全漏洞，并推进修复。根据 Anthropic 披露，一个隶属于阿尔伯塔省技术与创新部的团队，在二十小时内扫描了四点六六亿行代码，修复了系统中的安全缺口，还构建了一些新工具来提升系统安全性。

这条很值得注意，因为它不是一般意义上的 AI 写代码演示，而是一个政府部门在真实遗留系统里的大规模安全实践。政府系统通常承载福利发放、公共服务和敏感个人信息，但这些系统又经常存在代码老旧、文档不完整、安全债务积累的问题。Anthropic 把这个案例拿出来讲，本质上是在展示 Claude Code 不只是开发者的效率工具，也可以成为公共部门做系统治理和安全加固的基础设施。

我个人觉得，这类案例对行业的参考价值在于，它把 AI coding agent 的叙事从“帮程序员写一个函数”，推进到了“帮大型组织理解和修复庞大代码资产”。当然，安全修复本身仍然需要人类团队负责审查、验证和落地，但二十小时扫描四点六六亿行代码这个量级，确实说明 AI 在遗留系统安全排查上已经有非常明确的落点。

接下来，七月九号，Anthropic 宣布，Ben Bernanke 加入 Anthropic 的 Long-Term Benefit Trust。

Ben Bernanke 是前美联储主席，也是在二零二二年获得诺贝尔经济学奖的经济学家。他在二零零六年到二零一四年领导美联储，经历了二零零八年全球金融危机和之后的复苏阶段。现在他加入的 Long-Term Benefit Trust，是 Anthropic 的一个独立治理机构，职责是帮助公司坚持它的使命，也就是负责任地发展先进 AI，并让它长期造福人类。

这条属于公司治理动态，不是产品发布，但放在 Anthropic 的脉络里也有意义。Anthropic 一直强调自己是 Public Benefit Corporation，也一直强调长期风险、制度设计和公共利益。让一位长期研究金融危机、宏观经济和制度风险的人加入治理结构，某种程度上是在补强 Anthropic 对 AI 经济影响和系统性风险的判断能力。

同一天，Anthropic 还发布了一篇叫 Inviting hard questions 的文章，中文可以理解成“邀请大家提出那些难回答的问题”。

这篇文章不是模型能力更新，而更像是一篇面向公众的沟通稿。Anthropic 提到，人们对 AI 有很多期待，比如让工作和生活更轻松，改变学习方式，加速科学和技术进步，创造新的繁荣，甚至帮助解决医疗和社会问题。但同时，人们也有很多担忧，比如 AI 会不会导致失业，会不会贬低创造性工作，会不会影响人的自主性、人与人的连接，以及当强大 AI 落到错误的人手里时，风险是否会超过收益。

Anthropic 在这篇文章里再次强调了自己的公共利益使命，包括投入 AI 安全防护，降低滥用风险，研究模型行为和内部机制，帮助模型更好地对齐人类利益。它还提到通过影片呈现一些来自普通人的希望和担忧。

这条的重点不在技术细节，而在 Anthropic 的外部叙事。随着模型能力越来越强，AI 公司面对的已经不只是开发者和企业客户，还有监管者、教育工作者、创作者、家长和普通劳动者。Anthropic 选择把“难问题”摆到台面上，说明它在继续强化一个形象：不仅要做能力强的模型，也要参与关于 AI 规则、风险和社会影响的公共讨论。

再看第四条，也是七月九号，Claude 推出了一个 beta 版的新功能，用来帮助用户反思自己是怎么使用 Claude 的。

这个功能可以在 Claude 网页版或者桌面应用的设置里找到。它会汇总你使用 Claude 的情况，比如你经常聊哪些主题，使用模式是什么，通常拿 Claude 做哪些类型的任务。用户可以回看过去一个月、三个月、六个月或者十二个月的 Claude 聊天活动。它也会展示你最常在什么时候使用 Claude，以及那段时间主要在处理什么事情。Anthropic 还说，之后会加入一个视图，让用户看到自己使用 Claude 花了多少时间。

更有意思的是，这个功能不只是一个数据仪表盘。它还会周期性地抛出一些反思问题，比如，有什么事情即使 Claude 可以更快完成，你也仍然想自己做。用户还可以在面板里设置 quiet hours，也就是安静时段。

这条产品更新很小，但方向挺有意思。过去 AI 产品通常强调“更多使用、更高效率、更强能力”，而这个功能是在问另一个问题：你是不是在以符合自己目标的方式使用 AI。它把用户和 AI 的关系，从单纯的工具调用，稍微推进到使用习惯、边界感和自我管理上。对于开发者和产品经理来说，这也提示了一种新趋势：未来 AI 助手不只是帮你完成任务，也可能要帮你理解自己为什么、什么时候、以什么方式依赖 AI。

最后一条，七月九号，Anthropic 宣布和 UST 合作，把 Claude 带入 physical AI，也就是物理 AI 的场景。

这里的 physical AI，不是单纯指机器人，而是指把 AI 嵌入到生产设备、工程流程和现实产品制造链条里。比如芯片量产之前，要在晶圆厂里做压力测试；产品出货之前，装配线上的故障要尽早发现，避免最后变成召回事件。UST 是一家技术和工程服务公司，服务领域包括半导体、汽车、制造、电信、嵌入式系统和 IoT。它会把 Claude 用到客户依赖的工程环境里，并在全球范围内培训两万名工程师、架构师和顾问使用 Claude。

Anthropic 举的场景是，UST 帮客户构建和运行那些用来验证设计、验证芯片、运行工厂、以及服务已交付产品的系统。这类流程通常很长、步骤很多，而且越早发现问题，成本越低。一个设计缺陷如果在验证阶段被发现，可能只是工程师多花一个下午；但如果等到工厂已经进入制造阶段才发现，代价就可能是一整轮生产。

所以这条的意义在于，Claude 正在从知识工作、软件开发，继续往更硬的工程场景里走。它不只是写文档、写代码，也开始参与芯片、汽车、制造和连接设备背后的设计验证与生产流程。当然，这些场景对可靠性、可追溯性和责任边界的要求会更高，所以真正落地肯定不会是“模型自己决定一切”，而是进入现有工程系统，辅助工程师更早发现问题、更快处理复杂信息。

总结一下，这一周 Anthropic 没有发布新的大模型，但动作很集中：一边展示 Claude Code 在政府网络安全中的真实应用，一边把 Claude 推向工业工程和物理 AI，同时继续补强治理结构和公共沟通。最值得关注的，我个人会选阿尔伯塔省政府这个案例，因为它很清楚地说明，AI coding agent 的下一阶段战场，很可能是大型组织里那些复杂、陈旧但极其关键的代码系统。

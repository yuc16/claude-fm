## Script

大家好,欢迎收听 Anthropic 一周快讯,这次我们看的是二零二五年五月十八号到五月二十四号这一周。这周内容不多,一共两条,但分量都不轻,正好赶上 Anthropic 做了一次大动作,所以我们一条一条仔细说。

先说五月二十二号这一天,Anthropic 正式发布了新一代模型,一口气推出两款,Claude Opus 四和 Claude Sonnet 四。官方的说法是,Opus 四是目前世界上最好的编程模型,尤其擅长处理那种持续时间很长、步骤很多的复杂任务和智能体工作流,简单说就是让 Claude 自己规划、自己执行、长时间不掉链子地把一个大任务做完。Sonnet 四则是对上一代 Sonnet 三点七的一次显著升级,编程和推理能力都更强,而且对指令的理解也更精准,你让它做什么,它就更准确地照做,不容易跑偏。这次发布还带了几个值得关注的新东西:一个是扩展思考可以配合工具调用了,也就是模型在深度思考的过程中,可以一边推理一边去调用像网页搜索这样的外部工具,边想边查,这样回答质量会更高。另一个是模型现在支持并行调用多个工具,而且如果开发者给了它访问本地文件的权限,它的记忆能力会明显提升,能够主动提取关键信息存下来,长期积累、保持上下文的连续性,这对做长周期智能体应用的开发者来说是个挺实用的能力。还有一个大消息是 Claude Code 这次正式从研究预览版变成全面可用,之前内测的反馈一直不错,现在它支持通过 GitHub Actions 跑后台任务,还原生集成了 VS Code 和 JetBrains,编辑结果直接显示在你的代码文件里,等于是把结对编程这件事做得更顺手了。API 那边也同步开放了四项新能力,方便开发者基于这些新模型去搭建更复杂的应用。整体看,这次发布是 Anthropic 在编程和智能体方向上一次很扎实的迭代,如果你平时用 Claude Code 写代码或者在搭智能体工作流,这两款新模型基本是值得马上试一下的。

接下来这条和上面这条其实是同一天发生、紧密绑在一起的。还是五月二十二号,Anthropic 在正式部署 Claude Opus 四的同时,宣布启动了人工智能安全等级三,也就是 ASL 三的安全和部署标准,这是 Anthropic 负责任扩展政策里规定的一套机制。简单说,安全标准这部分主要是加强内部的安全防护,让模型权重更难被窃取;部署标准这部分则是针对一个很窄的风险点,就是防止 Claude 被滥用来帮助开发或获取化学、生物、放射性、核武器,也就是常说的 CBRN 风险。Anthropic 特别强调,这些限制措施只会让 Claude 在极少数特定话题上拒绝回答,不会影响日常使用。值得注意的是,他们说得很清楚,这次启用 ASL 三是一种预防性、临时性的谨慎做法,并不是说已经确认 Opus 四真的达到了需要这个等级保护的能力门槛,而是因为这代模型在 CBRN 相关知识和能力上持续在进步,以前那种可以直接排除风险的判断方式现在已经不够用了,需要更细致的研究才能下最终结论。同时他们也排除了 Opus 四需要更高等级也就是 ASL 四防护的可能,Sonnet 四则被确认不需要 ASL 三。这条快讯本身不涉及具体新功能,但我个人觉得它挺重要,因为这是 Anthropic 第一次在正式产品发布时同步拉高安全等级,说明随着模型能力越来越强,公司在风险评估和治理上也在同步收紧节奏,这对整个行业怎么看待前沿模型的安全部署,是一个值得记住的参照点。

整体来看,这周的核心就是 Claude 四这次发布,Opus 四和 Sonnet 四同时登场,Claude Code 全面可用,再叠加上一次首次启用的 ASL 三安全标准,技术能力和安全治理在同一天一起往前迈了一步。本周快讯就到这里,我们下期再见。

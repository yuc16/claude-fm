本期我们聊聊 Anthropic 在二〇二五年十月发布的一篇博客文章,主题是 Claude Code 的插件系统。文章不算长,但信息密度很高,讲的是怎么把斜杠命令、子代理、MCP 服务器和钩子这几种扩展能力打包成一个可以一键安装、一键分享的插件,还介绍了插件市场这个概念。结合现在已经是二〇二六年年中的时间点,我们也会聊聊这套机制在过去这大半年里给团队协作带来的实际变化。

本期要点：
- 插件到底是什么:它把斜杠命令、子代理、MCP 服务器、钩子这四种东西打包成一个可以开关的整体
- 为什么需要插件:之前这些能力都是分散配置的,团队很难统一标准,个人也很难把自己的好用配置分享出去
- 插件市场是什么:只需要一个 Git 仓库加一个 JSON 文件,任何人都能办一个市场,别人用一条命令就能添加
- 插件能开能关,目的是控制系统提示词的上下文体积和复杂度,不是装了就一直占着资源
- 社区已经跑出了一些有意思的案例,比如八十多个细分子代理的合集,以及面向开源维护者、工程负责人、框架作者的不同玩法
- 实践层面我们会聊聊普通开发者和团队负责人分别应该怎么上手,以及要避开哪些坑

## Shownotes（复制到小宇宙）

本期聊一篇来自 Anthropic 的博客文章,作者是 Claude Code 团队成员 Thariq Shihipar,他分享了为什么团队内部越来越多人放弃 Markdown,转而用 HTML 文件跟 Claude Code 协作产出方案、报告和设计稿。文章发表于 2026 年 5 月 20 日,核心观点是:当 Claude 能处理的任务越来越复杂、产出越来越长,HTML 比 Markdown 更能承载丰富信息,也更容易让人真正读完、愿意分享。我们拆解了文章里的核心论点,以及五个可以直接上手用的具体场景。

- Markdown 的瓶颈:超过一百行就没人愿意认真读,也难以呈现表格、配色、图示等信息
- HTML 的三大优势:信息表达力更强、可读性和分享性更好、可以做成可交互的调参画布
- 为什么用 Claude Code 而不是网页版生成 HTML:能调用本地代码、Git 历史、MCP 工具和浏览器上下文
- 五个具体用例:方案探索与设计选型、代码评审与 PR 讲解、设计原型与交互调参、跨数据源报告生成、一次性专用编辑器
- 常见疑问解答:HTML 是否更耗 token、是否完全抛弃 Markdown、这是否替代了"写计划"
- 主播个人点评:这套方法的真正价值是帮你在 AI 能力越来越强时保持"在场感",而不是沦为旁观者

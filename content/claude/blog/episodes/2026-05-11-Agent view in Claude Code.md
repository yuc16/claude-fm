本期我们聊聊 Claude Code 在二〇二六年五月十一日上线的新功能 agent view,智能体总览视图。这是一个专门用来管理多个并行 Claude Code 会话的界面,目标是让你不再需要靠一堆终端标签和心智账本去追踪每个任务的状态。我们会拆解它的交互设计、几种早期用户摸索出的实战用法,以及上手之前需要注意的几个坑。

本期要点
- agent view 是什么:一个集中查看所有 Claude Code 会话状态的总览面板,按左箭头或运行 claude agents 即可打开
- 瞄一眼加快速回复:不离开列表就能看到会话最后一轮内容并直接回答卡住的决策,按回车可进入完整对话
- 后台化能力:用 slash bg 把现有会话转入后台,或用 claude --bg 直接以后台模式启动新任务
- 早期用户的四种实战玩法:批量并发跑任务、管理长循环智能体、在主线和旁支任务间穿梭、快速扫描哪些任务已交付
- 当前定位与使用门槛:研究预览版,覆盖 Pro、Max、Team、Enterprise 和 Claude API 用户,正常速率限制依然适用
- 主播个人观察:这代表 AI 编程工具从单会话聊天转向多智能体协同管理的趋势

## Shownotes（复制到小宇宙）

本期我们聊一篇来自 Anthropic 官方博客的实战指南,主题是如何在 Claude Code 里用好最新的 Opus 四点七模型。文章核心是一句话:不要再把它当成需要你逐行带着走的搭档,而要把它当成一个可以放手托管任务的资深工程师。我们会拆解效力等级、自适应思考、以及一系列默认行为变化背后的逻辑,帮你少踩坑、多省 token。

本期要点
- Opus四点七在编码、企业工作流和长时任务上全面超越四点六,但 token 消耗模式也变了
- 交互式多轮对话和自主式单轮任务,模型的思考方式完全不同,理解这一点能省下大量等待时间
- 新的默认效力等级是 xhigh,介于 high 和 max 之间,官方建议大部分编码任务都用这一档
- 固定预算的 Extended Thinking 被废弃,换成了按步骤自主决定要不要多想的自适应思考
- 模型现在默认调用工具更少、生成子代理更少、回答长度更贴合任务复杂度,需要主动在提示词里说清楚你想要什么
- auto mode 进入研究预览阶段,配合一次性把任务说清楚,能显著缩短任务周转时间

先说一下时效性的问题。这篇文章发布于二〇二六年一月,今天是二〇二六年六月二十日,中间隔了差不多五个月。五个月在 AI 这个行业里不算短,但还没到半年,所以总体上这次更新的逻辑应该还没有过时。我们这期节目讲的是这次更新本身的设计思路,这些思路放到今天大概率还是成立的,具体数字大家以官网当下展示的为准就好。

我们先来搭一下背景。Claude 大家应该都不陌生,Anthropic 推出的对话式人工智能助手,从最早面向个人用户的免费版和订阅版,逐渐发展出了面向团队和企业的版本。在这次更新之前,Claude 已经有了好几档订阅,个人用户最常打交道的是 Pro 版,也就是个人专业版,一个人每月付费,获得比免费版更高的使用额度。而 Team 版,也就是我们今天的主角团队版,是面向多人协作场景设计的,简单理解,就是把原来一个人用的 Claude,变成一整个团队一起用,共享同一个工作空间,大家可以在同一个项目里协作,还能接入公司内部的知识库。

那为什么 Anthropic 要在二〇二六年初专门发一篇文章去调整团队版的定价呢。文章开篇第一句话说得很清楚,他们想让 Claude Team 套餐对正在成长中的团队更加友好,更容易负担。这句话信息量其实不小,关键词是成长中的团队。这说明这次调价瞄准的不是那些已经是巨头的大企业客户,而是那些规模还在扩张,人数从几个人涨到几十人、上百人的中小型团队,或者大公司里刚刚组建、还在试验阶段的小分队。这类团队的特点是,预算敏感,决策周期短,愿意尝试新工具,但同时对每一笔订阅支出都要精打细算。如果团队版的入门门槛太高,这些团队很可能会选择让每个人单独开一个 Pro 版个人账号,而不是上团队版,这样一来团队协作的功能,比如共享工作区、统一知识库接入,就都用不上了,大家各自为战,公司也很难统一管理和统一买单。

所以这次调价,本质上是 Anthropic 想降低团队版相对个人版的门槛,让更多团队心甘情愿地从零散的个人订阅,转向统一的团队订阅。这是典型的 to B 产品增长策略,用更低的入门价格换取更多团队整体迁移过来,迁移过来之后,无论是续费率还是席位扩张,都会比一盘散沙的个人订阅更稳定。这一点文章原文没有直接这么说,是我个人的解读,大家可以保留自己的判断。

好,我们来看这次调整最实在的部分,也就是具体的价格变化。Claude Team 套餐里有两种席位,一种叫标准席位,一种叫高级席位,英文分别是 standard seat 和 premium seat。

标准席位调整之后,如果按年付,价格是每个席位每月二十美元,如果选择按月付费,价格是每个席位每月二十五美元。年付和月付是两种计费方式,年付通常意味着要一次性签约一年,但平摊到每个月的单价更便宜,月付则更灵活,可以随时停,但单价贵一些,这是订阅制软件里非常常见的套路,健身房年卡和月卡的逻辑其实是一样的。

高级席位这次的变化更值得说一说。调整之后年付价格是每个席位每月一百美元,月付价格是每个席位每月一百二十五美元。文章里专门提到,这是高级席位第一次有年付折扣,也就是说在这次更新之前,高级席位可能压根没有年付优惠这个选项,大家只能按月付费,价格更贵也更没有弹性。现在 Anthropic 把这个优惠也开放给了高级席位,相当于把原来标准席位才有的福利,延伸到了价格更贵、用量更大的那一档上面。

我们稍微算一下账,方便大家有个直观感受。假设你的团队有二十个人,全部用标准席位,按年付计算,每人每月二十美元,二十个人就是每月四百美元,一年算下来是四千八百美元。如果这二十个人里有五个是重度用户,需要更大的用量,换成高级席位,按年付每人每月一百美元,五个人就是每月五百美元,剩下十五个标准席位每月三百美元,加起来团队每月支出八百美元,一年差不多是九千六百美元。这笔账具体准不准要看你团队真实的人数和用量需求,但至少能帮你建立一个大概的预算模型,跟财务或者老板申请预算的时候心里有个数。

价格只是故事的一半,另一半是你花这笔钱到底能用多少。文章里有一句话我觉得特别关键,原文说每一个团队版席位包含的使用量,都比 Pro 版个人版更多。也就是说,哪怕你用的是团队版里最便宜的标准席位,你能用的额度也比单独订阅一个 Pro 版个人账号要高。这对那些纠结要不要从个人订阅升级到团队版的用户来说,是个很有说服力的论据,价格差不太多,但用量给得更多,还多了团队协作的功能,怎么算都更划算。

再往上看,高级席位的用量是标准席位的五倍。这个倍数关系很直白,如果你团队里有那种每天疯狠用 Claude 写代码、跑分析、处理大量文档的重度用户,标准席位的额度可能撑不到月底就用完了,这时候高级席位的五倍用量就派上用场了。

那如果连高级席位的用量都不够用呢,文章里也给出了答案,管理员可以为团队额外购买容量。这意味着用量管理被拆成了两层,第一层是席位本身自带的基础额度,第二层是团队管理员可以按需加购的弹性容量池。这种设计我个人觉得挺聪明的,它避免了两种极端,一种是限额太死,团队稍微多用一点就报错卡住,影响正常工作,另一种是完全不限量,团队成本完全失控。现在这种分层设计,既给了基础保障,又给了管理员主动加码的空间,把成本控制的决策权交还给了团队自己。

对于负责采购和预算的同学,这里有个实际的启示,你不需要一开始就把所有人都开成高级席位去应对峰值用量,完全可以先给大部分人开标准席位,把少数重度用户单独升级到高级席位,如果还不够,再用管理员加购容量这个阀门去补,这样能把团队的整体支出控制得更精细,而不是简单粗暴地全员拉满。

聊完了钱,我们来看这次更新里另一个重点,就是 Anthropic 强调的团队版到底好在哪儿,文章里专门有一段在讲团队版是专为团队的协作方式而打造的。

第一个核心功能是共享工作空间。在团队版里,所有成员可以在同一个项目空间里和 Claude 协作。这跟个人版完全不是一个使用场景,个人版你一个人对着 Claude 聊,聊完的内容、上下文基本就锁在你自己的账号里,同事看不到也用不上。团队版把这个边界打开了,大家可以在同一个项目里看到彼此和 Claude 的互动历史,共享同一套上下文,团队里有人已经探索过的思路、试过的提示词、跑出来的结果,后面的人不用从零开始重新摸索,直接接着用就行。这其实解决了一个很现实的协作痛点,以前团队里每个人单独摸索 AI 工具的用法,效率参差不齐,知识没法在团队内部沉淀和传递,现在有了共享空间,这种沉淀就自然发生了。

第二个核心功能是连接器,英文是 connectors,可以接入公司内部的知识。这个功能我觉得是整篇文章里最值得工程师朋友们关注的一点。我们都知道,大语言模型本身的知识是训练数据截止时间之前的通用知识,它并不知道你公司内部的产品文档、过去的会议记录、内部的技术规范、客户反馈这些专属信息。如果每次都要手动把这些内部资料贴到对话框里再问 Claude,效率很低,而且很容易遗漏。连接器这个机制,本质上是给 Claude 接上了一根管子,让它能够直接读取团队授权给它的内部知识来源,这样团队成员问问题的时候,Claude 给出的回答就不再只是泛泛而谈的通用知识,而是结合了你们公司自己的业务上下文给出的针对性回答。这对于产品经理写需求文档、客服查历史工单、工程师查内部技术规范,都是实打实的效率提升。

第三个核心功能,也是我觉得最值得拿出来单独说一说的,是文章提到团队版的每一个席位都包含 Claude Code。Claude Code 是 Anthropic 推出的一款编程代理工具,简单理解就是一个可以在命令行或者开发环境里直接帮你写代码、改代码、跑测试、定位 bug 的智能体,它不只是聊天问答,而是能真正动手在你的代码仓库里干活。文章里特别强调,无论你是负责上线新功能的开发者,还是正在用自动化流程的产品经理,每个席位都包含 Claude Code。

我觉得这句话背后的意味值得多说几句。把 Claude Code 这种听起来很硬核的编程工具,同时开放给产品经理这种通常不写代码的角色,说明 Anthropic 想表达的是,编程能力不应该只属于工程师。你可以想象这样一个场景,产品经理想把每周手动整理的用户反馈自动汇总成报表,过去这种需求得排期找工程师写脚本,现在他可以直接用自然语言告诉 Claude Code 想要的效果,让它自己写代码、跑出来、迭代调整,整个过程完全不需要懂任何编程语法。这种把编程能力从工程师手里释放出来,交给更广泛角色使用的趋势,我个人认为是这两年 AI 行业最值得关注的变化之一,Claude Team 把 Claude Code 塞进每个席位,只是这个大趋势在产品层面的一个具体落地。

讲完了协作功能,我们再看团队版在管理层面的设计。文章提到团队版有集中化的管理员控制和账单功能。这句话听起来很平淡,但对于负责采购和 IT 管理的同学来说,这恰恰是决定能不能大规模推广的关键。试想一下,如果上百号员工都各自拿信用卡单独订阅,财务部门核对账单就得疯掉,而且没法统一管理权限,有人离职了账号是不是还在扣费,这些问题在分散订阅模式下几乎是失控的。团队版提供的统一管理后台,意味着管理员可以在一个地方添加成员、移除成员、查看用量,所有费用走一张账单,这对企业级的管理来说不是锦上添花,而是能不能落地的硬性要求。

数据隐私是另一个必须重点拎出来讲的点。文章原文说得很直接,默认情况下不会用你的内容去训练 Claude。这句话对企业客户分量很重,很多公司一开始对让员工把工作内容交给第三方 AI 工具是有顾虑的,顾虑的核心是,输入进去的商业机密、客户信息、内部代码,会不会被拿去训练模型。Anthropic 给出的承诺是默认不训练,默认这个词用得很关键,意味着这是默认设置而不是强制规定,如果团队主动同意,理论上是可以选择参与训练的,但前提是团队明确做出选择,而不是被动接受。这种默认保护加可选开启的设计,在面向企业的订阅软件里算是比较成熟稳妥的做法。

文章最后还有一个小小的行动号召,提到这些变化现在已经生效,如果想入手团队版,可以直接去官网开始,遇到问题官方帮助中心也有更多信息可以查。这种结尾在企业产品公告里很常见,基本就是告诉你别等了,现在就能用。

讲完了文章里的核心内容,我们花点时间聊聊这件事对你我这样的普通听众具体意味着什么,以及如果你要决定要不要把团队搬到 Claude Team 上,应该怎么想这件事。

第一个问题是,你的团队现在是不是已经有好几个人在各自单独订阅 Pro 版个人账号了。如果答案是是,那这次降价之后你应该立刻去重新算一遍账。我们前面说过,团队版标准席位年付的价格已经做到跟个人版差不太多的水平,但用量更高,还多了共享工作空间、知识连接器这些团队功能。换句话说,如果你团队里有三个人各自在用 Pro 版,你们大概率已经在为同样的钱获得更差的体验,因为彼此之间的上下文是隔离的,没法共享探索成果。这种情况下,迁移到团队版基本是个没什么悬念的决定,唯一需要确认的是具体席位数和当前真实价格,这个去官网核实一下就行。

第二个问题是,你的团队里有没有那种用量特别大的重度用户。这种人往往是团队里最早接受新工具、用得最狠的那一类,比如天天用 Claude Code 跑大段代码生成和重构的工程师,或者每天用 Claude 处理海量文档和数据分析的分析师。对这种人,直接给标准席位很可能撑不住,容易月底用量告警卡壳,影响他正常干活。这时候应该果断给他们配高级席位,五倍的用量基本能覆盖大部分重度场景,如果还不够,再考虑用管理员加购容量这个机制去兜底。反过来,团队里那些只是偶尔问问题、查查资料的轻度用户,标准席位就完全够用,没必要为了省事直接全员升到高级席位,那样反而是浪费预算。

第三个问题,是关于知识连接器这个功能怎么用得好。我建议大家在真正接入内部知识库之前,先想清楚两件事。第一件事是哪些内部资料是真正高频被问到、值得花精力接入的,不要一上来就想着把公司所有文档系统全部一股脑接进去,先从使用频率最高、最痛的场景切入,验证效果之后再逐步扩展。第二件事是权限和安全边界要提前规划好,哪些资料属于高度敏感、不适合直接暴露给所有团队成员通过 Claude 查询,这个在接入之前就应该跟你们的安全和合规团队对齐清楚,而不是接入之后再去补救。

第四个问题,关于 Claude Code 这件事,我想特别对产品经理和非工程背景的听众多说几句。如果你过去因为不会写代码,觉得 Claude Code 这种工具跟自己没关系,这次更新其实是在明确告诉你,它就是设计给你用的。我建议你可以从一个很小的自动化需求开始练手,比如每周要手动汇总一份数据报表,或者要把几个不同格式的文件批量转换成统一格式,这种重复性强、规则清晰的任务非常适合拿来当作你和 Claude Code 协作的第一个练习项目。不要一开始就挑战特别复杂的需求,从小任务开始,慢慢建立起对它能力边界的直觉,你会发现很多过去觉得必须排期找工程师帮忙的小事,自己现在就能搞定。

第五个问题,是给负责采购决策的管理者的建议。如果你正在评估是不是要把团队迁移到 Claude Team,我建议你不要只盯着价格表上的数字做决定,而是把价格、每个席位的用量、能不能加购弹性容量、管理后台好不好用、数据隐私政策是不是符合你们公司的合规要求,这几个维度综合权衡,才能做出真正适合你们团队规模和工作方式的决定。如果你们团队本身就处于快速扩张期,人数还在持续往上涨,我会格外建议你优先考虑团队版而不是让大家各自开个人账号,因为团队版的管理后台和统一账单,会在团队规模变大之后帮你省下大量的行政管理成本,这笔隐性的省钱账往往比订阅费本身的差价更大。

最后提一个稍微反方向的提醒。价格降了不代表要立刻给所有人升级到最高档位。比较稳妥的做法是先用一小部分团队成员试运行一段时间,真实评估用量和协作功能是否解决了你们的实际痛点,跑通了再考虑全员铺开,这样既能控制风险,也给团队成员留出适应和学习的时间。
